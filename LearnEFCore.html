EF Core Lab Entity Framework Core 8

Entity Framework Core — Two Approaches

A complete hands-on lab covering both Code First (you design C# classes, EF creates the database) and Database First (database exists first, EF generates your C# classes). Both use Visual Studio 2022 with .NET 8 and LocalDB.

🛠 Visual Studio 2022 📦 .NET 8.0 🗄 LocalDB (SQL Server) ⏱ ~2.5 hours
Before You Start

Code First vs Database First

Aspect Code First Database First
You createC# entity classesTables in SQL Server
EF generatesDatabase tables via migrationsC# entity classes via scaffold
Key commandAdd-Migration → Update-DatabaseScaffold-DbContext
Schema changesAdd C# property → run migrationALTER TABLE SQL → re-run scaffold
Who controls schemaDeveloper (C# code)DBA / SQL team
Best forNew greenfield projectsExisting databases
ℹ

Both approaches use the same EF Core runtime. The difference is only in HOW the database and C# classes are created. Once set up, querying, CRUD, and navigation properties work identically.

NuGet Packages — What Each One Does

Install these through Package Manager. Each step will tell you exactly which ones to install.

Package NameWhy you need itRequired?
Microsoft.EntityFrameworkCore Core EF library — DbContext, DbSet, LINQ translation. Pulled in automatically when you install the SqlServer package. You don't need to install this manually. Auto
Microsoft.EntityFrameworkCore.SqlServer SQL Server provider — enables UseSqlServer(). Also works with LocalDB. Install this one. It brings in the base package automatically. Required
Microsoft.EntityFrameworkCore.Design Required for migrations (Add-Migration, Update-Database) and scaffolding (Scaffold-DbContext). Without this, these commands fail. Required

Version must match your .NET version. For .NET 8, install version 8.0.x of all packages. Mixing versions (e.g. SqlServer 8.x with Tools 7.x) causes build errors.

Code First — Step 3
Create the Employee entity class
▼

🧠 What is an Entity?

An entity is a plain C# class that maps to a database table. EF Core reads your class and generates CREATE TABLE SQL automatically. Each property becomes a column. The naming convention Id or EmployeeId is automatically treated as the Primary Key.

1

Add a new file called Employee.cs

In Solution Explorer → right-click EFCodeFirst (the project) → Add → New Item → select C# Class → name it Employee.cs → click Add

2

Replace the entire content of Employee.cs with this code:

Employee.cs
C#
using System.ComponentModel.DataAnnotations;
using System.ComponentModel.DataAnnotations.Schema;

namespace EFCodeFirst;

// This class = one table in the database: dbo.Employees
// EF Core reads the properties and creates matching columns
public class Employee
{
    // int Id = EF Core auto-makes this the Primary Key
    // SQL Server assigns numbers automatically: 1, 2, 3...
    public int Id { get; set; }

    // [Required]   → NOT NULL column (cannot be empty)
    // [MaxLength]  → nvarchar(100) instead of nvarchar(max)
    [Required]
    [MaxLength(100)]
    public string Name { get; set; } = "";

    [Required]
    [MaxLength(150)]
    public string Email { get; set; } = "";

    [Required]
    [MaxLength(50)]
    public string Department { get; set; } = "";

    // [Column(TypeName)] → exact SQL Server column type
    // Without this, decimal defaults to decimal(18,2) but it's good practice to be explicit
    [Column(TypeName = "decimal(18,2)")]
    public decimal Salary { get; set; }

    public DateTime JoiningDate { get; set; }

    // IsActive = soft-delete flag
    // When we "delete" an employee, we set this to false
    // instead of physically removing the row
    public bool IsActive { get; set; } = true;

    // ToString() for easy Console.WriteLine output
    public override string ToString() =>
        $"[{Id}] {Name,-20} | {Department,-12} | ₹{Salary,8} | Active:{IsActive}";
}

Code First — Step 4
Create AppDbContext — your gateway to the database
▼

🧠 What is DbContext?

DbContext is your connection to the database. One DbSet<T> = one table. OnConfiguring is where you tell EF Core which database server and database to use. SaveChanges() sends all pending INSERT/UPDATE/DELETE in one transaction.

1

Add a new file called AppDbContext.cs (same way as Employee.cs)

Right-click project → Add → New Item → C# Class → AppDbContext.cs → Add

2

Paste this code into AppDbContext.cs:

AppDbContext.cs
C#
using Microsoft.EntityFrameworkCore;

namespace EFCodeFirst;

public class AppDbContext : DbContext
{
    // DbSet = represents the Employees table
    // EF Core generates: CREATE TABLE Employees (...)
    // Property name 'Employees' = table name in the database
    public DbSet<Employee> Employees { get; set; }

    protected override void OnConfiguring(DbContextOptionsBuilder options)
    {
        // UseSqlServer = connect to SQL Server / LocalDB
        // (localdb)\MSSQLLocalDB = LocalDB that ships with Visual Studio
        // Database=EFCodeFirstDB = EF Core creates this database if it doesn't exist
        // Trusted_Connection=True = use Windows Authentication (no username/password needed)
        // TrustServerCertificate=True = avoid SSL certificate warnings for LocalDB
        options.UseSqlServer(
            @"Server=(localdb)\MSSQLLocalDB;" +
            "Database=EFCodeFirstDB;" +
            "Trusted_Connection=True;" +
            "TrustServerCertificate=True;"
        );
    }
}

🧠 What is a Migration?

A migration is a versioned snapshot of your schema. EF Core reads your entity classes, compares them to the previous migration, and generates a C# file with Up() (apply changes) and Down() (undo changes) methods. Think of it as git for your database schema.

📦 Open Package Manager Console

  2. A console panel opens at the bottom of Visual Studio
  3. Make sure the Default project dropdown (top of the console) shows EFCodeFirst

Run this command in Package Manager Console:

Wait for it to complete. It may take 10–20 seconds the first time.

2

Confirm the Migrations folder was created in Solution Explorer:

Open the generated migration file — double-click 20250512xxxxxx_InitialCreate.cs in Solution Explorer and read it:

Generated Migration — read this, understand it
C#
// EF Core generated this entire file from your Employee.cs class
// You did not write any SQL — EF Core wrote it for you

protected override void Up(MigrationBuilder migrationBuilder)
{
    migrationBuilder.CreateTable(
        name: "Employees",
        columns: table => new
        {
            Id         = table.Column<int>(nullable: false)
                              .Annotation("SqlServer:Identity", "1, 1"),
            Name       = table.Column<string>(maxLength: 100, nullable: false),
            Email      = table.Column<string>(maxLength: 150, nullable: false),
            Department = table.Column<string>(maxLength: 50, nullable: false),
            Salary     = table.Column<decimal>(type: "decimal(18,2)"),
            JoiningDate= table.Column<DateTime>(),
            IsActive   = table.Column<bool>(defaultValue: true)
        },
        constraints: table => table.PrimaryKey("PK_Employees", x => x.Id)
    );
}

// Down() = UNDO this migration (drops the table)
protected override void Down(MigrationBuilder migrationBuilder)
{
    migrationBuilder.DropTable(name: "Employees");
}

This is version control for your database. Up() applies the change. Down() undoes it. EF Core tracks which migrations have been applied in dbo.__EFMigrationsHistory.

Code First — Step 7
Full CRUD in Program.cs — Add, Read, Update, Delete
▼
1

Open Program.cs and replace ALL content with this code:

Program.cs — Complete CRUD Demo
C#
using EFCodeFirst;

Console.WriteLine("=== EF Core — Code First Demo ===");
Console.WriteLine();

// 'using' ensures the database connection closes cleanly when done
using AppDbContext context = new AppDbContext();

// ── ADD (INSERT) ─────────────────────────────────────────────────
Console.WriteLine("Step 1 — Adding 3 employees...");

Employee emp1 = new()
{
    Name        = "Raushan Ranjan",
    Email       = "raushan@rrskillverse.com",
    Department  = "Training",
    Salary      = 85000m,
    JoiningDate = new DateTime(2020, 6, 1),
    IsActive    = true
};
Employee emp2 = new()
{
    Name        = "Priya Sharma",
    Email       = "priya@example.com",
    Department  = "Development",
    Salary      = 72000m,
    JoiningDate = new DateTime(2021, 3, 15),
    IsActive    = true
};
Employee emp3 = new()
{
    Name        = "Rahul Gupta",
    Email       = "rahul@example.com",
    Department  = "HR",
    Salary      = 60000m,
    JoiningDate = DateTime.Today,
    IsActive    = true
};

context.Employees.Add(emp1);   // queues INSERT — nothing sent yet
context.Employees.Add(emp2);
context.Employees.Add(emp3);
context.SaveChanges();         // NOW sends all 3 INSERTs to SQL Server

Console.WriteLine($"  ✓ emp1 added — Id={emp1.Id} (SQL Server assigned)");
Console.WriteLine($"  ✓ emp2 added — Id={emp2.Id}");
Console.WriteLine($"  ✓ emp3 added — Id={emp3.Id}");

// ── READ ALL (SELECT * FROM Employees) ───────────────────────────
Console.WriteLine();
Console.WriteLine("Step 2 — All employees:");
foreach (Employee e in context.Employees)
    Console.WriteLine("  " + e);

// ── FIND BY ID ───────────────────────────────────────────────────
Console.WriteLine();
Console.WriteLine("Step 3 — Find by Id:");
Employee? found = context.Employees.Find(emp2.Id);
Console.WriteLine("  Found: " + (found?.Name ?? "not found"));

// ── FILTER WITH LINQ ─────────────────────────────────────────────
Console.WriteLine();
Console.WriteLine("Step 4 — Training department only:");
var trainers = context.Employees
    .Where(e => e.Department == "Training" && e.IsActive)
    .OrderBy(e => e.Name)
    .ToList();
foreach (var e in trainers)
    Console.WriteLine("  " + e);

// ── UPDATE ────────────────────────────────────────────────────────
Console.WriteLine();
Console.WriteLine("Step 5 — Update Rahul's salary and department:");
Employee? rahul = context.Employees
    .FirstOrDefault(e => e.Name == "Rahul Gupta");
if (rahul != null)
{
    rahul.Salary     = 75000m;
    rahul.Department = "Operations";
    context.SaveChanges();   // UPDATE Employees SET Salary=..., Department=... WHERE Id=3
    Console.WriteLine("  Updated: " + rahul);
}

// ── SOFT DELETE (preferred in real apps) ─────────────────────────
Console.WriteLine();
Console.WriteLine("Step 6 — Soft delete Priya (mark inactive):");
if (found != null)
{
    found.IsActive = false;
    context.SaveChanges();   // UPDATE Employees SET IsActive=0 WHERE Id=2
    Console.WriteLine("  Deactivated: " + found.Name);
}

// ── READ ACTIVE ONLY ─────────────────────────────────────────────
Console.WriteLine();
Console.WriteLine("Step 7 — Active employees only:");
foreach (var e in context.Employees.Where(e => e.IsActive))
    Console.WriteLine("  " + e);

// ── HARD DELETE (physically remove) ──────────────────────────────
Console.WriteLine();
Console.WriteLine($"Step 8 — Hard delete Rahul (Id={emp3.Id}):");
Employee? toDelete = context.Employees.Find(emp3.Id);
if (toDelete != null)
{
    context.Employees.Remove(toDelete);
    context.SaveChanges();   // DELETE FROM Employees WHERE Id=3
    Console.WriteLine("  Deleted.");
}

// ── FINAL STATE ───────────────────────────────────────────────────
Console.WriteLine();
Console.WriteLine("Step 9 — Final state of database:");
foreach (var e in context.Employees)
    Console.WriteLine("  " + e);

Console.WriteLine();
Console.WriteLine("✓ Done. In SQL Server Object Explorer:");
Console.WriteLine("  Right-click dbo.Employees → View Data");

Code First — Step 8
Add a new column — the real migration workflow
▼

This is the most important Code First concept: you change C# → EF Core changes the database. No manual ALTER TABLE SQL needed.

1

Open Employee.cs and add this property after the IsActive property:

Employee.cs — add this property
C#
// New column — add this after IsActive
[MaxLength(20)]
public string? PhoneNumber { get; set; }

The ? after string means nullable — the column allows NULL. Existing rows won't break because PhoneNumber will be NULL for them.

2

In Package Manager Console, run a second migration:

Package Manager Console
PM Console
Add-Migration AddPhoneNumber
Update-Database
Expected
Applying migration '20250512xxxxxx_AddPhoneNumber'.
Done.
3

Verify in SQL Server Object Explorer → right-click dbo.Employees → View Designer. The PhoneNumber column now appears. Your existing data (Raushan and Priya) is still there — the column is NULL for them.

4

To roll back (undo the last migration):

Package Manager Console — rollback
PM Console
# Go back to the InitialCreate migration (removes PhoneNumber column)
Update-Database InitialCreate

# Delete the AddPhoneNumber migration file
Remove-Migration

This is how you fix mistakes — roll back to the last good migration, remove the bad file, fix your code, and try again. Your data is preserved through the rollback.

USE EFDatabaseFirstDB;
GO

-- Table 1: Departments (parent table)
CREATE TABLE Departments (
    Id        INT           IDENTITY(1,1) PRIMARY KEY,
    Name      NVARCHAR(100) NOT NULL,
    Location  NVARCHAR(100) NOT NULL
);
GO

-- Table 2: Employees (child table — has FK to Departments)
CREATE TABLE Employees (
    Id           INT           IDENTITY(1,1) PRIMARY KEY,
    Name         NVARCHAR(100) NOT NULL,
    Email        NVARCHAR(150) NOT NULL,
    Salary       DECIMAL(18,2) NOT NULL,
    JoiningDate  DATETIME2     NOT NULL,
    IsActive     BIT           NOT NULL DEFAULT 1,
    DepartmentId INT           NOT NULL,

    -- Foreign key: every Employee must belong to a Department
    CONSTRAINT FK_Employees_Departments
        FOREIGN KEY (DepartmentId)
        REFERENCES Departments(Id)
);
GO

-- Seed data: 3 departments
INSERT INTO Departments (Name, Location) VALUES
    ('Training',    'Noida'),
    ('Development', 'Bangalore'),
    ('HR',          'Delhi');
GO

-- Seed data: 3 employees (DepartmentId references the Departments rows above)
INSERT INTO Employees (Name, Email, Salary, JoiningDate, IsActive, DepartmentId) VALUES

🧠 What is Scaffolding?

Scaffold-DbContext is the Database First command. EF Core connects to your existing database, reads every table and relationship, and generates the matching C# entity classes and DbContext. You never write these classes manually — EF Core writes them for you.

1

Package Manager Console — Scaffold command
PM Console

  "Server=(localdb)\MSSQLLocalDB;Database=EFDatabaseFirstDB;Trusted_Connection=True;TrustServerCertificate=True;" `
  Microsoft.EntityFrameworkCore.SqlServer `
  -OutputDir Models `
  -ContextDir Data `
  -Context AppDbContext `
  -DataAnnotations `
  -NoOnConfiguring `
  -Force

Each flag explained:

-OutputDir ModelsEntity classes (Employee.cs, Department.cs) go in the Models\ folder
-ContextDir DataAppDbContext.cs goes in the Data\ folder
-Context AppDbContextName of the generated DbContext class
-DataAnnotationsUse [Required][MaxLength] attributes instead of Fluent API — easier to read
-NoOnConfiguringDon't hardcode the connection string in DbContext — we'll pass it in Program.cs
-ForceOverwrite existing generated files (useful when re-scaffolding after schema changes)

using System.ComponentModel.DataAnnotations;
using System.ComponentModel.DataAnnotations.Schema;

namespace EFDatabaseFirst.Models;

public partial class Department
{
    [Key]
    public int Id { get; set; }

    [Required]
    [StringLength(100)]
    public string Name { get; set; } = null!;

    [Required]
    [StringLength(100)]

Program.cs — Database First Demo
C#
using EFDatabaseFirst.Data;
using EFDatabaseFirst.Models;
using Microsoft.EntityFrameworkCore;

Console.WriteLine("=== EF Core — Database First Demo ===");
Console.WriteLine();

// Build DbContextOptions — connection string passed from here
// (We used -NoOnConfiguring in scaffold so it's not inside DbContext)
var options = new DbContextOptionsBuilder<AppDbContext>()
    .UseSqlServer(
        @"Server=(localdb)\MSSQLLocalDB;" +
        "Database=EFDatabaseFirstDB;" +
        "Trusted_Connection=True;" +
        "TrustServerCertificate=True;"
    )
    .Options;

using AppDbContext context = new AppDbContext(options);

// ── READ seeded data ─────────────────────────────────────────────
Console.WriteLine("Step 1 — All employees from seeded database:");
foreach (Employee e in context.Employees)
    Console.WriteLine($"  [{e.Id}] {e.Name,-20} | ₹{e.Salary,8} | Active:{e.IsActive}");

// ── JOIN — employees WITH their department names ──────────────────
Console.WriteLine();
Console.WriteLine("Step 2 — Employees with Department (JOIN using .Include):");
var withDept = context.Employees
    .Include(e => e.Department)     // LEFT JOIN Departments ON DepartmentId
    .OrderBy(e => e.Name)
    .ToList();

foreach (var e in withDept)
    Console.WriteLine($"  {e.Name,-20} | Dept: {e.Department?.Name,-15} | {e.Department?.Location}");

// ── GROUP — departments with employee count ───────────────────────
Console.WriteLine();
Console.WriteLine("Step 3 — Departments with active employee count:");
var deptSummary = context.Departments
    .Include(d => d.Employees)
    .Select(d => new
    {
        d.Name,
        d.Location,
        ActiveCount = d.Employees.Count(e => e.IsActive)
    })
    .ToList();

foreach (var d in deptSummary)
    Console.WriteLine($"  {d.Name,-15} | {d.Location,-12} | Active: {d.ActiveCount}");

// ── ADD — new employee via C# ─────────────────────────────────────
Console.WriteLine();
Console.WriteLine("Step 4 — Add new employee (Rahul, Training dept):");
Employee newEmp = new()
{
    Name         = "Rahul Gupta",
    Email        = "rahul@example.com",
    Salary       = 60000m,
    JoiningDate  = DateTime.Today,
    IsActive     = true,
    DepartmentId = 1   // Training (Id=1 from seed data)
};
context.Employees.Add(newEmp);
context.SaveChanges();
Console.WriteLine($"  ✓ Added: [{newEmp.Id}] {newEmp.Name}");

// ── UPDATE ────────────────────────────────────────────────────────
Console.WriteLine();
Console.WriteLine("Step 5 — Give Rahul a salary raise:");
Employee? rahul = context.Employees.FirstOrDefault(e => e.Name == "Rahul Gupta");
if (rahul != null)
{
    rahul.Salary = 75000m;
    context.SaveChanges();
    Console.WriteLine($"  Updated salary: ₹{rahul.Salary}");
}

// ── SOFT DELETE ───────────────────────────────────────────────────
Console.WriteLine();
Console.WriteLine("Step 6 — Soft delete Rahul:");
if (rahul != null)
{
    rahul.IsActive = false;
    context.SaveChanges();
    Console.WriteLine($"  Deactivated: {rahul.Name}");
}

// ── FINAL READ — active employees with JOIN ───────────────────────
Console.WriteLine();
Console.WriteLine("Step 7 — Active employees with department:");
var active = context.Employees
    .Include(e => e.Department)
    .Where(e => e.IsActive)
    .OrderBy(e => e.Name)
    .ToList();

foreach (var e in active)
    Console.WriteLine($"  {e.Name,-20} | {e.Department?.Name,-15} | ₹{e.Salary}");

Console.WriteLine();
Console.WriteLine("✓ Done — check dbo.Employees in SQL Server Object Explorer → View Data");
