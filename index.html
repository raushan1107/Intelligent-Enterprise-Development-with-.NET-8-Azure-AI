MODULE 01
Writing Applications in C# and .NET 8
.NET 8 SDKCLRVisual Studio 2022dotnet CLI
.NET 8 is Microsoft's latest Long-Term Support (LTS) release — a cross-platform, open-source successor that unified the .NET Framework and .NET Core lineages. The Common Language Runtime (CLR) is its execution engine: it compiles your C# source into platform-neutral Intermediate Language (IL), then JIT-compiles that IL to native CPU instructions at runtime. The CLR also manages memory (Garbage Collector), enforces type safety, and handles threads. Understanding this pipeline explains every performance decision you'll make.
💡 Real-World Analogy
Think of .NET 8 as a modern factory. Your C# code is the blueprint (IL), the CLR is the assembly-line manager who translates blueprints into actual work (native code), the GC is the clean-up crew, and Visual Studio is your engineering workstation. NuGet is the parts warehouse.
Topics Covered
🌐
.NET 8 Ecosystem — SDK (build tools + runtime), Runtime (CLR + BCL), NuGet (package registry). Evolution: .NET Framework → .NET Core → .NET 5→8 LTS. Key improvement in .NET 8: Native AOT, frozen collections, primary constructors.
⚙️
CLR & JIT — Compilation stages: C# → Roslyn compiler → IL bytecode → JIT → native. Tiered compilation: Tier0 (quick JIT) → Tier1 (optimised JIT after hot path detection). The GC uses generational collection: Gen0, Gen1, Gen2 + LOH.
📁
Project Structure — .csproj (MSBuild XML), Program.cs with top-level statements (no Main boilerplate since C# 9), obj/ and bin/ folders, .sln solution file grouping multiple projects.
⌨️
dotnet CLI — dotnet new console, dotnet build, dotnet run, dotnet publish -c Release. Self-contained vs framework-dependent deployments.
🏗️
Visual Studio 2022 — Solution Explorer, IntelliSense, live code analysis, integrated debugger with breakpoints/watch/step-over, hot reload for instant feedback without restart.
Code Reference
C# · Program.cs
// .NET 8 — top-level statements (no class/Main required) // dotnet new console -n ERP.Enrollment --framework net8.0 Console.WriteLine("🏫 ERP Enrollment System v1.0"); var appName = "ERP Core"; // inferred as string const int MAX_BATCH = 30; // compile-time constant Console.WriteLine($"App: {appName} | Max batch: {MAX_BATCH}"); // CLR manages memory — GC reclaims this automatically var list = new List<string> { "Alice", "Bob", "Riya" }; foreach (var name in list) Console.WriteLine($" ↳ Student: {name}"); // dotnet CLI commands reference: // dotnet new console -n MyApp // dotnet build ← compiles to IL // dotnet run ← JIT → native → execute // dotnet publish -c Release ← optimised distribution
MODULE 02
Types of Data and Expressions
Value TypesReference TypesNullableType Conversion
C# is a statically and strongly typed language — every variable has a type fixed at compile time. The key distinction is where data lives in memory: value types live on the stack (fast, copied on assignment), reference types live on the heap (a pointer is copied). Getting this wrong causes subtle bugs — especially with collections and DTO mapping in ERP systems.
💡 Real-World Analogy
Value type = writing a number on a sticky note and handing someone a copy — they can change their copy without affecting yours. Reference type = handing someone the key to the same filing cabinet — both of you see the same documents, and changes affect everyone who has a key.
Topics Covered
📦
Value Types — int, long, double, decimal, bool, char, struct, enum. Stored directly on the stack. Default values are zero/false/empty. Use decimal for financial calculations (avoids floating-point rounding in ERP invoicing).
🔗
Reference Types — string, class, array, interface, delegate. Variable holds a heap address. Assignment copies the reference, not the object. string is immutable — concatenation in a loop creates many objects (use StringBuilder).
🔄
Type Conversion — Implicit (widening: int → long), Explicit cast ((int)3.9 = 3), Convert.ToInt32(), int.Parse() (throws on failure), int.TryParse() (safe, returns bool).
❓
Nullable Types — int? = Nullable<int>. Has .HasValue and .Value. Null-coalescing: x ?? 0. Null-conditional: student?.Name. Critical for DB columns that allow NULL in EF Core models.
➕
Operators & Expressions — Arithmetic, relational, logical (&&, ||), bitwise, ternary condition ? a : b, switch expression, pattern matching with is.
Code Reference
C# · Types
// Value vs Reference — copy behaviour int a = 10; int b = a; // copy of value b = 20; // a stays 10 var s1 = new Student { Name = "Alice" }; var s2 = s1; // both point to SAME object s2.Name = "Bob"; // s1.Name is now "Bob" too! // Nullable — critical for EF Core columns int? score = null; int finalScore = score ?? 0; // 0 if null string? email = student?.Email; // safe navigation // Use decimal (NOT double) for money! decimal feeAmount = 15000.75m; // 'm' suffix = decimal literal double wrong = 0.1 + 0.2; // = 0.30000000000000004 ❌ // Safe parsing — never use Parse() on user input string input = Console.ReadLine()!; if (int.TryParse(input, out int age)) Console.WriteLine($"Age: {age}"); else Console.WriteLine("Invalid age entered.");
MODULE 03
C# Language Constructs
ConditionsLoopsException HandlingPattern Matching
Control flow determines which code runs and how many times. C# 10/11 introduced powerful switch expressions and pattern matching that replace verbose if-else chains. Combined with structured exception handling, these constructs let you write readable, resilient business logic — essential for an enrollment validation pipeline.
Topics Covered
🔀
Conditions — if/else if/else, switch statement, switch expression (C# 8+) with arm syntax, pattern matching using is, type patterns, relational patterns (>= 90 => "A"), and when guards.
🔁
Loops — for (index-based), while (condition-based), do-while (always runs once), foreach (iterates IEnumerable). break exits the loop, continue skips the current iteration. Avoid infinite loops — always have an exit condition.
💥
Exception Handling — try wraps risky code, catch handles specific exception types (most specific first), finally always runs (use for cleanup like closing files/DB connections). throw re-throws preserving stack trace; throw ex resets it (avoid).
Code Reference
C# · Control Flow
// Switch expression — grades in an ERP system string GetGrade(int marks) => marks switch { >= 90 => "A+", >= 80 and < 90 => "A", >= 70 => "B", >= 60 => "C", >= 40 => "D", _ => "F" // discard = default }; // foreach with break/continue — process enrollments var students = new[] { "Alice", "Bob", "", "Riya" }; foreach (var name in students) { if (string.IsNullOrWhiteSpace(name)) continue; // skip blanks Console.WriteLine($"Enrolling: {name}"); } // Structured exception handling try { int batchSize = int.Parse(Console.ReadLine()!); if (batchSize <= 0) throw new ArgumentOutOfRangeException( nameof(batchSize), "Batch size must be positive."); Console.WriteLine($"Processing {batchSize} students..."); } catch (FormatException) { Console.WriteLine("Please enter a valid number."); } catch (ArgumentOutOfRangeException ex) { Console.WriteLine($"Validation error: {ex.Message}"); } finally { Console.WriteLine("Session ended."); // always runs }

MODULE 04
Methods & Overloading
Method DesignOverloadingparamsref/out
Methods are the primary unit of code reuse in C#. A well-designed method does one thing (SRP), has a descriptive name, and returns a meaningful value. Method overloading lets you offer multiple call signatures under the same name, reducing the caller's cognitive load — the same pattern used by Console.WriteLine() itself (it has 18 overloads).
💡 Real-World Analogy
A bank's Transfer() service is overloaded: you can call Transfer(amount), Transfer(amount, currency), or Transfer(amount, currency, scheduledDate). Each overload handles a different level of detail — but the intent is always the same. The bank doesn't create three different service desks; one desk handles all variants.
Topics Covered
🔧
Defining Methods — access modifier (public, private, internal), return type (void or typed), static vs instance, expression body (=>) for single-line methods.
📥
Parameters — value (copy), ref (two-way reference), out (guaranteed assignment from callee), in (read-only ref, no copy), params (variable argument count), optional parameters with defaults.
🔁
Overloading Rules — same name, different parameter count or types. Return type alone cannot differentiate overloads. Compiler resolves the best match at compile time using specificity rules.
♻️
Refactoring for Reusability — DRY (Don't Repeat Yourself), extracting private helpers, using optional parameters to consolidate logic, expression-bodied members for brevity.
Code Reference
C# · Methods
public class FeeCalculator { // Expression-bodied — single return public decimal Calculate(decimal base_) => base_ * 1.18m; // Overload 1 — with scholarship public decimal Calculate(decimal base_, decimal scholarship) => (base_ - scholarship) * 1.18m; // Overload 2 — with instalment count public decimal Calculate(decimal base_, int instalments) => base_ / instalments; // params — accepts any number of discount codes public decimal ApplyDiscounts(decimal fee, params decimal[] discounts) => fee - discounts.Sum(); // out — parse + validate in one call public bool TryParseRollNo(string input, out int rollNo) { rollNo = 0; return int.TryParse(input, out rollNo) && rollNo > 0; } }
MODULE 05
Exception Handling & Logging
Custom ExceptionsILoggerSerilogStructured Logging
Production software fails — the question is whether it fails gracefully and observably. Custom exception classes convey domain semantics ("this student is already enrolled") far better than generic Exception. Structured logging (Serilog / Microsoft.Extensions.Logging) writes machine-parseable JSON logs that Azure Monitor, Application Insights, and Seq can query, alert on, and visualise.
Topics Covered
💥
Exception Filters — catch (SqlException ex) when (ex.Number == 1205) handles deadlocks specifically without catching other SQL errors. Exception filters don't unwind the stack, preserving the original context for debugging.
🏷️
Custom Exceptions — Inherit from Exception or domain-specific bases. Always include: default constructor, message constructor, inner-exception constructor (serialization in .NET 8 is automatic with primary constructors).
📝
Structured Logging — Use message templates ("Enrolled {StudentId} into {CourseCode}") not string interpolation. Template tokens become queryable properties in log management systems. Log levels: Trace, Debug, Information, Warning, Error, Critical.
🔭
Scopes & Correlation — _logger.BeginScope() attaches context (e.g., RequestId, StudentId) to all log entries within a block. Essential for tracing a single user's action across multiple service calls in a distributed ERP.
Code Reference
C# · Exceptions + Logging
// Custom domain exception with extra context public class EnrollmentException(string studentId, string courseCode, string msg) : Exception(msg) { public string StudentId { get; } = studentId; public string CourseCode { get; } = courseCode; } public class EnrollmentService(ILogger<EnrollmentService> logger) { public void Enroll(string studentId, string courseCode) { // Scope attaches studentId to ALL logs inside this block using var scope = logger.BeginScope( "Student={StudentId} Course={CourseCode}", studentId, courseCode); logger.LogInformation("Enrollment attempt started"); try { ValidateCapacity(courseCode); // may throw PersistEnrollment(studentId, courseCode); logger.LogInformation("Enrollment successful"); } catch (EnrollmentException ex) { logger.LogWarning(ex, "Business rule violation: {Reason}", ex.Message); throw; // preserve stack trace } } }

MODULE 06
Structs & Enums
structrecord structenumDomain Modeling
Structs are value types — ideal for small, immutable data with no identity (coordinates, RGB colour, money amount). Enums replace magic numbers/strings in domain code, making intent explicit. In ERP models, an EnrollmentStatus enum prevents invalid states like status = "apporved" ever reaching the database.
Topics Covered
🧱
struct vs class — Use struct when: <16 bytes, immutable, no inheritance needed, frequently allocated on the stack (e.g., DateTime, Guid, decimal). record struct (C# 10) adds value equality and with expressions.
🏷️
Enums in Domain Modeling — Assign explicit integer values for DB persistence stability. Use [Flags] attribute for bit-field combinations (e.g., Permissions.Read | Permissions.Write). Enum.Parse<T>() and Enum.TryParse<T>() for deserialization.
Code Reference
C# · Structs & Enums
// Enum — enrollment states in ERP public enum EnrollmentStatus { Pending = 0, Active = 1, Completed = 2, Withdrawn = 3, Suspended = 4 } // record struct — immutable money value public readonly record struct Money(decimal Amount, string Currency) { public Money Add(Money other) { if (Currency != other.Currency) throw new InvalidOperationException("Currency mismatch"); return this with { Amount = Amount + other.Amount }; } public override string ToString() => $"{Currency} {Amount:N2}"; } // Usage var enrollment = new Enrollment { Status = EnrollmentStatus.Active }; var fee = new Money(15000m, "INR"); Console.WriteLine(fee); // INR 15,000.00
MODULE 07
Built-in Collections & LINQ
List<T>DictionaryIEnumerableLINQDeferred Execution
LINQ (Language Integrated Query) brings SQL-like expressiveness directly into C#. It works over any IEnumerable<T> source — in-memory lists, EF Core DbSets, XML documents, or custom data sources — using the same query operators. Understanding deferred execution (the query runs only when iterated) is crucial for EF Core performance: adding a .ToList() in the wrong place can pull thousands of rows into memory.
Topics Covered
📋
List<T> — dynamic array, O(1) amortized add, O(n) insert/remove. .Add(), .Remove(), .Sort(), .BinarySearch(). Pre-set capacity to avoid re-allocations: new List<T>(1000).
📖
Dictionary<TKey,TValue> — hash-table backed, O(1) average lookup. Always use TryGetValue() over direct indexer to avoid KeyNotFoundException. ConcurrentDictionary<> for thread-safe scenarios.
⚡
LINQ Operators — Filtering: Where. Projection: Select, SelectMany. Ordering: OrderBy, ThenBy. Grouping: GroupBy. Joining: Join, GroupJoin. Aggregation: Count, Sum, Average, Min, Max. Materialisation: ToList(), ToArray(), ToDictionary().
⏳
Deferred Execution — LINQ queries are recipes, not results. They execute when enumerated. This means you can chain operators cheaply. But never iterate a query twice in EF Core — call .ToList() once and work with the list.
Code Reference
C# · LINQ
// ERP scenario: analyse student performance var students = GetStudents(); // returns List<Student> // Chained LINQ — still a query, not yet executed var atRisk = students .Where(s => s.GPA < 2.0m && s.Status == EnrollmentStatus.Active) .OrderBy(s => s.GPA) .Select(s => new { s.Name, s.GPA, s.AdvisorEmail }); // .ToList() executes the query — ONE database round-trip var results = atRisk.ToList(); // GroupBy — students by department with averages var deptReport = students .GroupBy(s => s.Department) .Select(g => new { Department = g.Key, Count = g.Count(), AvgGPA = g.Average(s => s.GPA), TopStudent = g.MaxBy(s => s.GPA)?.Name }) .OrderByDescending(d => d.AvgGPA) .ToList(); // Dictionary lookup — O(1) vs O(n) list search var courseMap = courses.ToDictionary(c => c.Code, c => c); if (courseMap.TryGetValue("CS101", out var course)) Console.WriteLine($"Found: {course.Name}");
MODULE 08
Events & Delegates
delegateAction/FunceventEventArgs
Delegates are type-safe function pointers — variables that hold references to methods. Events are delegates with restricted access (only the owner can invoke them; subscribers can only add/remove handlers). This publisher-subscriber pattern decouples components: an EnrollmentService raises OnEnrolled; an email service, a reporting service, and a notification service all subscribe independently — none aware of the others.
Topics Covered
📡
Delegates — delegate void Handler(string msg). Built-in generics: Action<T> (void return), Func<T,TResult> (with return), Predicate<T> (bool). Multicast delegates chain multiple methods with +=.
📢
Events — event EventHandler<EnrolledEventArgs> OnEnrolled. Subscribers add handlers with +=, remove with -=. Convention: raise via protected virtual method OnSomething() for subclass override. Always null-check before invocation: OnEnrolled?.Invoke(this, args).
🧩
Lambda Expressions — Anonymous methods inline: students.Where(s => s.GPA > 3.0). Closures capture outer variables — be careful with loop variables in closures.
Code Reference
C# · Events
public class EnrolledEventArgs(string studentId, string courseCode) : EventArgs { public string StudentId { get; } = studentId; public string CourseCode { get; } = courseCode; } public class EnrollmentService { // Event declaration — subscribers can only += or -= public event EventHandler<EnrolledEventArgs>? StudentEnrolled; public void Enroll(string studentId, string courseCode) { // ... persist enrollment ... // Raise the event — null-safe invocation StudentEnrolled?.Invoke(this, new EnrolledEventArgs(studentId, courseCode)); } } // Subscribers — completely decoupled from each other var svc = new EnrollmentService(); svc.StudentEnrolled += (_, e) => Console.WriteLine($"📧 Email sent to {e.StudentId}"); svc.StudentEnrolled += (_, e) => Console.WriteLine($"📊 Report updated for course {e.CourseCode}");
MODULE 09
Creating Classes & Interfaces
OOPInterfaceSOLIDAbstraction
Interfaces define contracts without implementations. When a class implements IEnrollmentRepository, any code that depends on the interface can work with a real EF Core repository, a test double, or an in-memory stub — without changes. This is the foundation of Dependency Injection and testable architecture. SOLID principles are the five rules that keep OOP systems maintainable.
Topics Covered
🏛️
Class Design — Properties (auto-implemented, computed), constructors, primary constructors (C# 12), methods, access modifiers. sealed prevents inheritance; abstract forces it.
📜
Interfaces — Define method signatures, properties, events. A class can implement multiple interfaces (unlike inheritance). default interface methods (C# 8) add implementations without breaking existing consumers.
🔑
SOLID Principles — Single Responsibility (one reason to change), Open/Closed (open for extension, closed for modification), Liskov Substitution (subtypes usable in place of base), Interface Segregation (small focused interfaces), Dependency Inversion (depend on abstractions, not concretions).
Code Reference
C# · Interfaces & SOLID
// Interface — the contract (no implementation) public interface IStudentRepository { Task<Student?> GetByIdAsync(string id); Task AddAsync(Student student); Task<IEnumerable<Student>> GetAllAsync(); } // Concrete implementation (EF Core, SQL) public class EfStudentRepository(ErpDbContext db) : IStudentRepository { public async Task<Student?> GetByIdAsync(string id) => await db.Students.FindAsync(id); public async Task AddAsync(Student student) { await db.Students.AddAsync(student); await db.SaveChangesAsync(); } public async Task<IEnumerable<Student>> GetAllAsync() => await db.Students.ToListAsync(); } // Service depends on INTERFACE — not concrete class (DIP) public class EnrollmentService(IStudentRepository repo) { public async Task EnrollAsync(Student s) => await repo.AddAsync(s); }

MODULE 10
Inheritance & Polymorphism
abstractvirtual/overridesealedPolymorphism
Inheritance lets a class reuse and extend another's behaviour. Polymorphism lets you treat different subtypes uniformly through a base reference — the correct method is selected at runtime. In an ERP, a Report base class can be subclassed into EnrollmentReport, GradeReport, and FinancialReport; all can be rendered by the same renderer that only knows Report.
Topics Covered
🧬
Inheritance — class Dog : Animal. base keyword calls parent constructor or method. Single inheritance in C# (multiple via interfaces). protected members accessible in subclasses. Avoid deep inheritance hierarchies (prefer composition over inheritance).
🎭
Polymorphism — virtual marks a method as overridable. override in subclass. abstract forces subclasses to implement. sealed override prevents further overriding. Runtime dispatch: which method runs is determined by the actual object type, not the variable type.
🚫
Custom Exception Hierarchies — ErpException (base) → EnrollmentException, FeeException, InventoryException. Catch at the right level of specificity. Never catch Exception and swallow it silently.
Code Reference
C# · Inheritance
public abstract class Report { public string Title { get; set; } = ""; public abstract string Generate(); // must be implemented public virtual string Footer() // may be overridden => $"Generated: {DateTime.Now:dd-MMM-yyyy}"; } public class EnrollmentReport(IEnumerable<Student> students) : Report { public override string Generate() => $"Enrollment Report\nTotal: {students.Count()}\n{Footer()}"; } public class GradeReport(IEnumerable<Grade> grades) : Report { public override string Generate() => $"Grade Report\nAvg: {grades.Average(g => g.Score):F1}\n{Footer()}"; public override string Footer() // customised footer => "Confidential — Academic Records"; } // Polymorphic rendering — same code, different outputs Report[] reports = { new EnrollmentReport(students), new GradeReport(grades) }; foreach (var r in reports) Console.WriteLine(r.Generate());
MODULE 11
Input & Output — File I/O & JSON
File I/OStreamsSystem.Text.JsonSerialization
System.IO provides file, directory, and stream operations. System.Text.Json (built into .NET) is the recommended serialiser — fast, allocation-efficient, and supports source generators for AOT-safe scenarios. JSON serialisation/deserialisation is the bridge between your C# domain objects and REST API payloads, configuration files, and persistent storage.
Topics Covered
📂
File & Directory API — File.ReadAllText(), File.WriteAllTextAsync(), Directory.GetFiles(), Path.Combine(). Always use Path.Combine() — never string concatenation for paths (OS separator differences).
🌊
Streams — FileStream, StreamReader, StreamWriter. Use await using to ensure disposal. MemoryStream for in-memory byte operations (e.g., serving file downloads from API).
🔄
JSON Serialization — JsonSerializer.Serialize(obj, options), JsonSerializer.DeserializeAsync<T>(stream). Configure with JsonSerializerOptions: camelCase, null handling, enums as strings, indented output.
Code Reference
C# · File I/O & JSON

MODULE 12
Generics in C#
Generic ClassesConstraintsGeneric RepositoryCovariance
Generics enable type-safe, reusable code without boxing/unboxing overhead. List<T>, Dictionary<TKey,TValue>, and Task<T> are all generic. The Generic Repository Pattern eliminates duplicate CRUD code — one Repository<T> class works for Student, Course, Grade, and any other entity.
Topics Covered
🔷
Generic Classes & Methods — class Stack<T>, T Swap<T>(ref T a, ref T b). Type parameter is resolved at compile time — no runtime casting, full IntelliSense.
🔒
Constraints — where T : class (reference type), where T : struct, where T : new() (has parameterless constructor), where T : IEntity (implements interface). Constraints unlock members of T inside the generic body.
🗄️
Generic Repository — IRepository<T> with GetByIdAsync, AddAsync, UpdateAsync, DeleteAsync. One EF Core implementation covers all entities. Specific repos inherit and add domain methods: IStudentRepository : IRepository<Student>.
Code Reference
C# · Generics
public interface IRepository<T> where T : class { Task<T?> GetByIdAsync(int id); Task<IList<T>> GetAllAsync(); Task AddAsync(T entity); Task UpdateAsync(T entity); Task DeleteAsync(int id); } // One implementation — works for ALL entity types public class EfRepository<T>(ErpDbContext db) : IRepository<T> where T : class { private readonly DbSet<T> _set = db.Set<T>(); public async Task<T?> GetByIdAsync(int id) => await _set.FindAsync(id); public async Task<IList<T>> GetAllAsync() => await _set.ToListAsync(); public async Task AddAsync(T e) { await _set.AddAsync(e); await db.SaveChangesAsync(); } public async Task UpdateAsync(T e) { _set.Update(e); await db.SaveChangesAsync(); } public async Task DeleteAsync(int id) { var e = await _set.FindAsync(id); if (e is not null) { _set.Remove(e); await db.SaveChangesAsync(); } } } // DI registration — one line covers all entities // builder.Services.AddScoped(typeof(IRepository<>), typeof(EfRepository<>));

Database Access with EF Core
3 modules · Estimated 7 hours · Lab: ERP Database + CRUD
MODULE 13
Entity Framework Core 8
DbContextMigrationsFluent APIRelationships
EF Core is a code-first ORM — you define C# classes, EF generates the SQL schema via migrations. DbContext is the gateway to the database: it tracks entity state changes and generates optimised SQL. Understanding the change tracker and migration snapshots prevents the most common pitfalls: N+1 queries, concurrency conflicts, and failed migrations in production.
Topics Covered
🗄️
DbContext — Unit of Work pattern. Each DbSet<T> maps to a table. SaveChangesAsync() commits all tracked changes in one transaction. Register as Scoped in DI (one per HTTP request).
🔄
Migrations — dotnet ef migrations add InitialCreate creates a snapshot diff. dotnet ef database update applies it. Never edit migration files manually — create a new migration for corrections. Use migration bundles for CI/CD deployments.
⚙️
Fluent API — OnModelCreating() configures entities without attributes: index uniqueness, composite keys, table/column names, check constraints, owned entities (value objects).
🔗
Relationships — One-to-Many: HasMany().WithOne(). Many-to-Many: explicit join entity in EF Core 5+. Cascade delete modes: Cascade, Restrict, SetNull. Always load related data explicitly with .Include() — EF Core does not lazy-load by default.

MODULE 17
Authentication & Authorization
JWTClaimsRole-basedPolicy-based
JWT (JSON Web Token) is the industry standard for stateless API authentication. A token consists of three Base64-encoded parts: Header (algorithm), Payload (claims), Signature (tamper proof). The server verifies the signature on every request — no session store needed. Claims are key-value pairs inside the token: user ID, roles, department, permissions — all available without a DB lookup.
Topics Covered
🔑
JWT Structure — header.payload.signature. Signing: HMAC-SHA256 (symmetric) or RS256 (asymmetric, ideal for microservices). Access token (short-lived: 15min–8h) + Refresh token (long-lived, stored securely).
👤
Claims — Standard: sub (subject/user ID), name, email, role, exp (expiry), iat (issued at). Custom: department_id, tenant_id for multi-tenant ERP.
🛡️
Role-based Auth — [Authorize(Roles = "Admin,Manager")]. Multiple roles are OR — user needs any one of them. Role hierarchy implemented via claims (e.g., Admin role includes Manager permissions).
📋
Policy-based Auth — Custom requirements + handlers: MinimumGPARequirement, SameDepartmentRequirement. Registered via AddPolicy(). More flexible than roles for complex business rules.

MODULE 18
API Security Best Practices
OWASPRate LimitingBCRYPTSecurity Logging
The OWASP API Security Top 10 describes the most critical risks for REST APIs. An ERP system is a high-value target — it contains PII, financial records, and business-critical data. Security is not a feature added at the end; it must be designed in from Day 1. This module covers the practical defences every enterprise API must have.
Topics Covered
⚠️
OWASP API Top 10 — API1: Broken Object Level Auth (check ownership per request), API2: Broken Auth (no weak tokens), API3: Broken Object Property Level Auth (never mass-assign), API4: Unrestricted Resource Consumption (rate limit), API5: BFLA (Broken Function Level Auth — check roles on every endpoint).
🔐

🚦
Rate Limiting — Microsoft.AspNetCore.RateLimiting (built-in .NET 8). Fixed window, sliding window, token bucket, concurrency limiters. Apply globally and per-endpoint. Return 429 Too Many Requests with Retry-After header.
📋
Security Event Logging — Log all authentication failures (with IP), authorization denials, admin actions, and data exports. Use correlation IDs. Forward to Azure Sentinel or Application Insights for alerting.

Overview of Blazor

Blazor is Microsoft's framework for building interactive web UIs using C# instead of JavaScript. .NET 8 introduced the Unified Rendering Model: a single app can mix Server-Side Rendering (SSR), Blazor Server (SignalR), and Blazor WebAssembly components — each page or component chooses its render mode. Topics Covered
🖥️
Blazor Server — UI runs on the server; browser sends events via SignalR, receives DOM diffs. Pros: fast initial load, full .NET access, small download. Cons: latency-sensitive interactivity, server memory per user.
🌐
Blazor WebAssembly — .NET runtime compiled to WASM runs in the browser. True client-side SPA. Pros: offline capable, no server dependency. Cons: large initial download, limited to browser sandbox.
🔗
Unified Rendering (.NET 8) — @rendermode InteractiveServer, @rendermode InteractiveWebAssembly, or @rendermode InteractiveAuto (WASM when available, Server as fallback). SSR (static) for non-interactive pages — fastest perceived load.
⚡
Lazy Loading — Load assembly DLLs on first navigation to a route area. Reduces initial WASM bundle size. Configure in Router component with OnNavigateAsync.

public record AnalyzeStudentQuery(int StudentId) : IRequest<AiAnalysisResult>; public class AnalyzeStudentHandler( ErpDbContext db, IAiService ai) : IRequestHandler<AnalyzeStudentQuery, AiAnalysisResult> { public async Task<AiAnalysisResult> Handle(AnalyzeStudentQuery q, CancellationToken ct) { var grades = await db.Grades .Where(g => g.StudentId == q.StudentId) .OrderByDescending(g => g.ExamDate) .Take(10) .Select(g => new { g.Subject, g.Score, g.ExamDate }) .ToListAsync(ct); var prompt = $""" Analyse these grades for student {q.StudentId}: {JsonSerializer.Serialize(grades)} Respond ONLY with valid JSON (no markdown): {{ "riskLevel": "Low|Medium|High", "trend": "Improving|Declining|Stable", "recommendations": ["action1", "action2", "action3"] }} """; var json = await ai.CompleteAsync(prompt, "You are an academic advisor AI."); return JsonSerializer.Deserialize<AiAnalysisResult>(json)!; } }
