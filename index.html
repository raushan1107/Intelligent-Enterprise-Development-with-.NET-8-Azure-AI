MODULE 01
Writing Applications in C# and .NET 8
.NET 8 SDKCLR / JITdotnet CLIVS 2022
.NET 8 is Microsoft's latest LTS runtime — a unified, cross-platform successor to .NET Framework and .NET Core. Your C# source compiles to platform-neutral Intermediate Language (IL); the CLR's JIT converts it to native machine code at runtime. The CLR also manages memory via the Garbage Collector and handles threading. Understanding this pipeline explains every performance decision in the course.
💡 Real-World Analogy
Think of .NET 8 as a modern manufacturing plant. C# code is the engineering blueprint (IL), the CLR is the assembly-line manager that converts blueprints to product (native code), the GC is the clean-up crew, and Visual Studio is your engineering workstation. NuGet is the parts warehouse.
Topics Covered
🌐
.NET 8 Ecosystem — SDK, Runtime, NuGet. Evolution: .NET Framework → .NET Core → .NET 5→8 LTS. New in .NET 8: Native AOT, frozen collections, primary constructors, time abstractions.
⚙️
CLR & JIT Pipeline — C# → Roslyn → IL → JIT → native code. Tiered compilation: Tier0 (fast JIT) → Tier1 (optimised hot-path JIT). GC generations: Gen0, Gen1, Gen2, LOH.
📁
Project Structure — .csproj SDK-style, top-level Program.cs (no boilerplate Main), obj/ intermediates, bin/ assemblies, .sln grouping.
⌨️
dotnet CLI — dotnet new console, dotnet build, dotnet run, dotnet publish -c Release --self-contained.
🔥
Hot Reload & Debugging — Apply code changes without restart. Breakpoints, Watch, Immediate Window, call stack, conditional & data breakpoints.
Code Reference
C# · Program.cs (top-level statements)
// .NET 8 — no class or Main boilerplate needed Console.WriteLine("🏫 ERP Enrollment System v1.0"); var appName = "ERP Core"; const int MAX = 30; // compile-time constant — no heap allocation var roster = new List<string> { "Alice", "Bob", "Riya" }; foreach (var name in roster) Console.WriteLine($" ↳ {name}"); // dotnet build → IL in /bin // dotnet run → JIT → native → execute // dotnet publish → optimised deployable output
MODULE 02
Types of Data and Expressions
Value TypesReference TypesNullableType Conversion
C# is statically and strongly typed. The key distinction: value types live on the stack (copied on assignment), reference types live on the heap (pointer is copied). Getting this wrong is the root of many subtle ERP bugs — especially in collection mutations and DTO mapping.
💡 Real-World Analogy
Value type = handing someone a photocopy of a document — they can scribble on theirs without changing yours. Reference type = giving someone the key to the same filing cabinet — any change either of you makes is visible to both.
Topics Covered
📦
Value Types — int, long, decimal, bool, char, struct, enum. Stack-stored. Always use decimal for money — double accumulates floating-point error (0.1 + 0.2 ≠ 0.3 in binary).
🔗
Reference Types — string, class, array, interface. Variable holds a heap address. string is immutable — use StringBuilder for repeated concatenation in loops.
🔄
Type Conversion — Implicit widening (int → long). Explicit cast ((int)3.9 = 3). Always use int.TryParse() for user input — int.Parse() throws on invalid input.
❓
Nullable Types — int? = Nullable<int>. Null-coalescing: score ?? 0. Null-conditional: student?.Address?.City. Essential for EF Core nullable columns.
➕
Pattern Matching — Switch expression, is type patterns, is not null, relational patterns (>= 90).
Code Reference
C# · Types, Nullability & Conversion
int a = 10; int b = a; // value copy — independent b = 20; // a is still 10 var s1 = new Student { Name = "Alice" }; var s2 = s1; // both point to SAME heap object s2.Name = "Bob"; // s1.Name is now "Bob" too — classic bug! decimal fee = 15000.75m; // 'm' suffix = decimal literal double wrong = 0.1 + 0.2; // = 0.30000000000000004 ❌ never for money int? score = null; int final = score ?? 0; // null-coalescing fallback string? city = student?.Address?.City; // safe null chain if (int.TryParse(Console.ReadLine()!, out int age) && age is >= 15 and <= 60) Console.WriteLine($"Valid age: {age}"); else Console.WriteLine("Invalid — must be 15–60.");
MODULE 03
C# Language Constructs
ConditionsLoopsSwitch ExpressionException Handling
Control flow determines which code runs and how many times. C# 10/11's switch expressions and pattern matching replace verbose if-else chains with declarative, compile-time-checked logic — the backbone of every business rule in the enrollment pipeline.
Topics Covered
🔀
Switch Expression — Arm syntax (pattern => result), relational patterns (>= 90 => "A"), discard default (_ => "F"). Exhaustiveness checked at compile time.
🔁
Loops — for, while, do-while, foreach. break exits; continue skips; return exits the method.
💥
Exception Handling — try/catch/finally. Catch most-specific first. finally always runs — use for cleanup. throw re-throws preserving stack trace; throw ex resets it (avoid).
🎯
Pattern Matching — is type: if (obj is Student s). Property: s is { GPA: > 3.5 }. List pattern (C# 11): [var first, ..].
Code Reference
C# · Switch Expression + Exception Handling

MODULE 04
Methods & Overloading
Overloadingparamsref / outDRY Design
Methods are the primary unit of code reuse. A well-designed method does one thing. Method overloading lets multiple signatures share one name — the same pattern Console.WriteLine() uses across 18 overloads. Compose overloads so they all delegate to one canonical implementation — one place to fix bugs.
💡 Real-World Analogy
A bank's Transfer desk is overloaded: "transfer ₹5000", "transfer ₹5000 in USD", "transfer ₹5000 on 1st of next month" — same service desk, same intent, different details. No separate counter for each variant.
Topics Covered
🔧
Defining Methods — Access modifier + return type + name + params + body. Expression-bodied (=>) for single-expression methods. Local functions for private helpers scoped inside a method.
📥
Parameter Modifiers — ref: two-way (modifies caller's variable). out: guaranteed assignment. params: variable-count args. in: read-only reference for large structs.
🔁
Overloading Rules — Same name, different parameter count or types. Return type alone does not differentiate. Compiler selects the most specific match.
Code Reference
C# · Method Overloading
public class FeeCalculator { public decimal Calculate(decimal base_) => base_ * 1.18m; // GST included public decimal Calculate(decimal base_, decimal scholarship) => (base_ - scholarship) * 1.18m; public decimal Calculate(decimal base_, int instalments) => Calculate(base_) / instalments; // delegates to base public decimal ApplyDiscounts(decimal fee, params decimal[] discounts) => fee - discounts.Sum(); } var calc = new FeeCalculator(); Console.WriteLine(calc.Calculate(15000m)); // 17700.00 Console.WriteLine(calc.Calculate(15000m, 2000m)); // 15340.00 Console.WriteLine(calc.Calculate(15000m, 3)); // 5900.00
MODULE 05
Exception Handling & Structured Logging
Custom ExceptionsILoggerSerilogLog Scopes
Production software fails — the question is whether it fails gracefully and observably. Custom exception classes carry domain semantics. Structured logging writes machine-parseable JSON (Serilog, Microsoft.Extensions.Logging) that Azure Monitor and Application Insights can query, alert on, and trace — letting you reconstruct exactly what happened before any production failure.
Topics Covered
🏷️
Custom Exception Classes — Inherit from Exception. Primary constructors (C# 12) make this a one-liner. Add domain properties (StudentId, CourseCode) for richer diagnostics.
🎯
Exception Filters — catch (SqlException ex) when (ex.Number == 1205). Filters don't unwind the stack — the original call context is preserved for debugging.
📝
Structured Logging — Message templates with named holes: "Enrolled {StudentId} in {CourseCode}". Tokens become queryable log properties. Log levels: Trace → Debug → Information → Warning → Error → Critical.
🔭
Log Scopes — logger.BeginScope() attaches ambient context (RequestId, StudentId) to every entry inside the block — essential for tracing across distributed services.
Code Reference
C# · Custom Exceptions + Structured Logging
public class EnrollmentException(string studentId, string courseCode, string msg) : Exception(msg) { public string StudentId { get; } = studentId; public string CourseCode { get; } = courseCode; } public class EnrollmentService(ILogger<EnrollmentService> logger) { public void Enroll(string studentId, string courseCode) { using var scope = logger.BeginScope( "Student={StudentId} Course={CourseCode}", studentId, courseCode); try { CheckCapacity(courseCode); Persist(studentId, courseCode); logger.LogInformation("Enrollment succeeded"); } catch (EnrollmentException ex) { logger.LogWarning(ex, "Business rule violation: {Reason}", ex.Message); throw; // preserves original stack trace } } }

MODULE 06
Structures & Enumerations
structrecord structenumDomain Modelling
Structs are value types ideal for small, immutable data with no identity. record struct (C# 10) adds value equality and with expressions automatically. Enums replace magic numbers in domain code — preventing invalid states like status = "approvd" from ever reaching the database.
Topics Covered
🧱
struct vs class — Use struct when ≤ 16 bytes, immutable, no inheritance. readonly struct prevents mutation. record struct (C# 10) gives value equality and with expressions.
🏷️
Enums in Domain Modelling — Explicit DB-stable integer values: Pending = 0, Active = 1. [Flags] for bitfields. Enum.TryParse<T>() for safe string/int deserialisation.
🔄
With-Expressions — Non-destructive copy: fee with {{ Amount = 12000m }}. Original unchanged — key pattern for immutable value objects in DDD.
Code Reference
C# · Structs & Enums
public enum EnrollmentStatus { Pending=0, Active=1, Completed=2, Withdrawn=3 } public readonly record struct Money(decimal Amount, string Currency) { public Money Add(Money other) { if (Currency != other.Currency) throw new InvalidOperationException("Cannot add different currencies"); return this with { Amount = Amount + other.Amount }; } public override string ToString() => $"{Currency} {Amount:N2}"; } var fee1 = new Money(15000m, "INR"); var fee2 = new Money(15000m, "INR"); Console.WriteLine(fee1 == fee2); // true — value equality var updated = fee1 with { Amount = 12000m }; // fee1 unchanged
MODULE 07
Built-in Collections & LINQ
List<T>DictionaryLINQDeferred Execution
LINQ brings SQL-style expressiveness to any IEnumerable<T>. The critical concept: deferred execution — LINQ queries are recipes, not results. The query runs only when enumerated. Adding .ToList() in the wrong place in EF Core pulls thousands of rows into memory.
Topics Covered
📋
List<T> — Dynamic array, O(1) amortised append. Pre-set capacity: new List<Student>(1000). .Sort(), .Find(), .RemoveAll().
📖
Dictionary<TKey, TValue> — O(1) average lookup. Always use .TryGetValue() to avoid KeyNotFoundException. ConcurrentDictionary<> for thread-safe caches.
⚡
LINQ Operators — Where, Select, SelectMany, OrderBy, GroupBy, Join, Count, Sum, Average, MinBy, MaxBy. Materialise with ToList(), ToDictionary(), ToHashSet().
⏳
Deferred Execution — Never iterate an EF Core IQueryable twice — each enumeration is a separate DB round-trip. Call .ToList() once and work with the result.
Code Reference
C# · LINQ Analytics
var atRisk = students .Where(s => s.GPA < 2.0m && s.Status == EnrollmentStatus.Active) .OrderBy(s => s.GPA) .Select(s => new { s.Name, s.GPA, s.AdvisorEmail }); var results = atRisk.ToList(); // executes here — ONE round-trip var report = students .GroupBy(s => s.Department) .Select(g => new { Department = g.Key, Count = g.Count(), AvgGPA = g.Average(s => s.GPA), TopStudent = g.MaxBy(s => s.GPA)?.Name }) .OrderByDescending(d => d.AvgGPA) .ToList(); var idx = courses.ToDictionary(c => c.Code); if (idx.TryGetValue("CS101", out var course)) Console.WriteLine($"Seats: {course.AvailableSeats}");
MODULE 08
Events & Delegates
delegateAction / FunceventPublisher-Subscriber
Delegates are type-safe function pointers. Events are delegates with restricted access — only the owner raises them. This publisher-subscriber pattern decouples components: EnrollmentService raises StudentEnrolled; an email service, reporting service, and notification service all subscribe independently — none aware of the others.
Topics Covered
📡
Delegates — Built-in: Action<T> (void), Func<T, TResult>, Predicate<T>. Multicast via +=.
📢
Events — event EventHandler<TEventArgs>. Subscribers += / -=. Always null-check before invoke: StudentEnrolled?.Invoke(this, args).
🧩
Lambdas & Closures — Anonymous inline methods. Closures capture enclosing scope. Be careful with loop-variable capture — use a local copy inside the lambda.
Code Reference
C# · Publisher-Subscriber Pattern
public class EnrolledEventArgs(string studentId, string courseCode) : EventArgs { public string StudentId { get; } = studentId; public string CourseCode { get; } = courseCode; } public class EnrollmentService { public event EventHandler<EnrolledEventArgs>? StudentEnrolled; public void Enroll(string sid, string code) { // ... persist ... StudentEnrolled?.Invoke(this, new EnrolledEventArgs(sid, code)); } } var svc = new EnrollmentService(); svc.StudentEnrolled += (_, e) => Console.WriteLine($"📧 Email → {e.StudentId}"); svc.StudentEnrolled += (_, e) => Console.WriteLine($"📊 Dashboard → {e.CourseCode}"); svc.StudentEnrolled += (_, e) => Console.WriteLine($"💬 WhatsApp dispatched");
MODULE 09
Creating Classes & Interfaces
OOPInterfaceSOLIDDependency Inversion
Interfaces define contracts without implementation. When EnrollmentService depends on IStudentRepository instead of EfStudentRepository, any consumer can swap in a real EF Core repo, an in-memory test stub, or a future Cosmos DB implementation — without touching the service. This is the foundation of Dependency Injection and testable enterprise architecture.
Topics Covered
🏛️
Class Design — Auto-properties, computed properties, primary constructors (C# 12), access modifiers. sealed prevents inheritance. abstract forces subclasses to implement.
📜
Interfaces — Method/property/event signatures. A class can implement multiple interfaces. Enables polymorphism without inheritance. Default implementations in C# 8+.
🔑
SOLID — Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation, Dependency Inversion. Applied throughout every subsequent module.
Code Reference
C# · Interface & DIP
public interface IStudentRepository { Task<Student?> GetByIdAsync(int id); Task AddAsync(Student student); Task<IReadOnlyList<Student>> GetAllAsync(); } public class EfStudentRepository(ErpDbContext db) : IStudentRepository { public async Task<Student?> GetByIdAsync(int id) => await db.Students.FindAsync(id); public async Task AddAsync(Student s) { await db.Students.AddAsync(s); await db.SaveChangesAsync(); } public async Task<IReadOnlyList<Student>> GetAllAsync() => await db.Students.ToListAsync(); } // Service depends on INTERFACE — not the concrete class (DIP) public class EnrollmentService(IStudentRepository students) { public async Task EnrollAsync(Student s) => await students.AddAsync(s); } // Swap impl in DI without touching service: // builder.Services.AddScoped<IStudentRepository, EfStudentRepository>();

MODULE 10
Inheritance & Polymorphism
abstractvirtual / overridesealedRuntime Dispatch
Polymorphism lets you treat subtypes uniformly through a base reference — the runtime selects the correct method. In the ERP, a Report base class is subclassed into EnrollmentReport, GradeReport, and FinancialReport; one renderer handles all three without knowing their specifics.
Topics Covered
🧬
Inheritance — class GradeReport : Report. base calls parent constructor or method. Single class inheritance in C#. Prefer composition (HAS-A) over deep IS-A trees.
🎭
virtual / abstract / sealed — virtual: can be overridden. abstract: must be overridden (no base body). sealed override: no further overriding. Runtime dispatch: actual object type determines which method runs.
💥
Exception Hierarchies — ErpException → EnrollmentException, FeeException. Catch at the right specificity level. Never swallow silently.
Code Reference
C# · Abstract + Polymorphism
public abstract class Report { public string Title { get; set; } = ""; public abstract string Generate(); // subclass MUST implement public virtual string Footer() // subclass MAY override => $"Generated: {DateTime.Now:dd-MMM-yyyy HH:mm}"; } public class GradeReport(IEnumerable<Grade> grades) : Report { public override string Generate() => $"Grade Avg: {grades.Average(g => g.Score):F1}\n{Footer()}"; public override string Footer() => "CONFIDENTIAL — Academic Records"; } // Polymorphic loop — one foreach handles ALL report types Report[] reports = [new EnrollmentReport(students), new GradeReport(grades)]; foreach (var r in reports) Console.WriteLine(r.Generate());

File / StreamSystem.Text.Jsonasync/awaitIAsyncDisposable
System.IO handles files, directories, and streams. System.Text.Json (.NET 8 built-in) is the recommended serialiser — fast, AOT-safe, allocation-efficient. JSON is the bridge between C# domain objects and REST payloads, config files, and persistent storage. Always use async stream APIs to avoid blocking threads on large files.
Topics Covered
📂

Streams — FileStream, StreamReader/Writer. Use await using for IAsyncDisposable. MemoryStream for in-memory byte buffers (serving file downloads from API without touching disk).

JSON Serialisation — JsonSerializer.SerializeAsync/DeserializeAsync<T>(). Configure: camelCase, null handling, enums as strings, indented output.

MODULE 12
Generics in C#
Generic ClassesConstraintsGeneric RepositoryOpen Generics DI
Generics enable type-safe, reusable code without boxing overhead. The Generic Repository Pattern eliminates duplicate CRUD code — one EfRepository<T> covers Student, Course, Grade, and every future entity. Register as an open generic in DI and all entities are covered with one line.
Topics Covered
🔷
Generic Classes & Methods — class Stack<T>. Type resolved at compile time — no runtime casting, full IntelliSense. Avoids boxing (value type → object) which allocates heap.
🔒
Constraints — where T : class, where T : struct, where T : new(), where T : IEntity. Constraints unlock T's members inside the generic body.
🗄️
Open Generics in DI — AddScoped(typeof(IRepository<>), typeof(EfRepository<>)) — covers all entities with a single registration line.
Code Reference
C# · Generic Repository
public interface IRepository<T> where T : class { Task<T?> GetByIdAsync(int id); Task<IList<T>> GetAllAsync(); Task AddAsync(T entity); Task UpdateAsync(T entity); Task DeleteAsync(int id); } // One class covers ALL entity types public class EfRepository<T>(ErpDbContext db) : IRepository<T> where T : class { private readonly DbSet<T> _set = db.Set<T>(); public async Task<T?> GetByIdAsync(int id) => await _set.FindAsync(id); public async Task<IList<T>> GetAllAsync() => await _set.ToListAsync(); public async Task AddAsync(T e) { await _set.AddAsync(e); await db.SaveChangesAsync(); } public async Task UpdateAsync(T e) { _set.Update(e); await db.SaveChangesAsync(); } public async Task DeleteAsync(int id) { if (await _set.FindAsync(id) is T e) { _set.Remove(e); await db.SaveChangesAsync(); } } } // builder.Services.AddScoped(typeof(IRepository<>), typeof(EfRepository<>));

MODULE 13
Entity Framework Core 8
DbContextMigrationsFluent APIRelationships
EF Core is a code-first ORM — you define C# classes and EF generates the SQL schema via migrations. DbContext is the Unit of Work: it tracks entity state and generates optimised SQL. Understanding the change tracker and migration snapshots prevents the most common pitfalls: N+1 queries, concurrency conflicts, and failed production migrations.
Topics Covered
🗄️
DbContext — Each DbSet<T> maps to a table. SaveChangesAsync() commits all tracked changes in one transaction. Register as Scoped (one per HTTP request — never Singleton).
🔄
Migrations — dotnet ef migrations add diffs model vs snapshot. dotnet ef database update applies it. Never hand-edit migration files. Use migration bundles for CI/CD.
⚙️
Fluent API — OnModelCreating(): unique indexes, composite keys, table names, owned entities, table-per-hierarchy inheritance, check constraints.
🔗
Relationships — One-to-Many, Many-to-Many (explicit join entity). Cascade delete modes. Always use .Include() — EF Core 8 doesn't lazy-load by default.
Code Reference
C# · EF Core 8 DbContext
public class ErpDbContext(DbContextOptions<ErpDbContext> opts) : DbContext(opts) { public DbSet<Student> Students { get; set; } public DbSet<Course> Courses { get; set; } public DbSet<Enrollment> Enrollments { get; set; } protected override void OnModelCreating(ModelBuilder mb) { mb.Entity<Student>().HasIndex(s => s.Email).IsUnique(); mb.Entity<Enrollment>() .HasOne(e => e.Course).WithMany(c => c.Enrollments) .HasForeignKey(e => e.CourseId).OnDelete(DeleteBehavior.Restrict); mb.Entity<Student>().OwnsOne(s => s.Address); // owned value object } } // dotnet ef migrations add InitialCreate --project ERP.Infrastructure // dotnet ef database update
MODULE 14
LINQ & Data Querying with EF Core
AsNoTrackingProjectionsEager LoadingCompiled Queries
LINQ over EF Core translates C# to SQL. The #1 performance rule: filter server-side, project to DTOs, materialise once. AsNoTracking() for read-only queries eliminates change-tracker overhead — 20–40% memory saving in reporting with thousands of rows.
Topics Covered
🚀
AsNoTracking() — Skips change tracker. Use on every GET endpoint. Critical at scale: with 10,000 tracked entities, SaveChanges() scans all of them even for unrelated updates.
📦
Projections — .Select(s => new StudentDto{...}) generates SELECT name, email not SELECT *. Never return full entities on read-only list views.
🔗
Eager Loading — .Include().ThenInclude() generates a JOIN — prevents N+1. .AsSplitQuery() for large datasets to avoid Cartesian explosion.
⚡
Compiled Queries — EF.CompileAsyncQuery() pre-compiles LINQ-to-SQL translation. Skip the translation overhead on hot-path search endpoints.
Code Reference
C# · EF Core Queries
var summaries = await db.Students .AsNoTracking() .Where(s => s.Status == EnrollmentStatus.Active && s.DepartmentId == deptId) .Include(s => s.Enrollments).ThenInclude(e => e.Course) .Select(s => new StudentSummaryDto { Id = s.Id, Name = s.FullName, CourseCount = s.Enrollments.Count(), AvgGrade = s.Enrollments.Average(e => (double?)e.Grade) ?? 0 }) .OrderBy(s => s.Name) .ToListAsync(ct); // Compiled query — pre-translates on first call, reused after private static readonly Func<ErpDbContext, int, Task<Student?>> GetById = EF.CompileAsyncQuery((ErpDbContext ctx, int id) => ctx.Students.Include(s => s.Enrollments).FirstOrDefault(s => s.Id == id));

MiddlewareDI ContainerConfigurationMinimal APIs
ASP.NET Core is a pipeline of middleware — each piece processes the request and passes it forward. Program.cs configures the pipeline and the DI container. Dependency Injection is the backbone of testable .NET 8 apps: services are registered once, injected anywhere.

Middleware Pipeline — Request flows forward through middleware in order; response flows back in reverse. Built-in: HTTPS redirect, static files, auth, routing, CORS, rate limiting.

MODULE 16
Building RESTful APIs
ControllersDTOsVersioningProblemDetails
REST uses HTTP verbs semantically: GET=read, POST=create, PUT=replace, PATCH=partial, DELETE=remove. Topics Covered
🗺️
Routing & Binding — Attribute routing with version: [Route("api/v{v:apiVersion}/[controller]")]. [FromBody], [FromRoute], [FromQuery]. [ApiController] auto-returns 400 on ModelState failure.
📦
DTOs as Records — Separate CreateStudentRequest, UpdateStudentRequest, StudentResponse. Immutable records. Manual mapping extensions (ToDto(), ToEntity()) for performance-critical paths.
💥

MODULE 17
Authentication & Authorisation with JWT
JWTClaimsRole-basedPolicy-based
JWT is the industry standard for stateless API auth. A token has three Base64-encoded parts: Header (algorithm), Payload (claims), Signature (tamper-proof). The server verifies the signature on every request — no session store needed. Claims carry user identity and permissions inside the token without any database lookup per request.
Topics Covered
🔑
JWT Structure — header.payload.signature. HMAC-SHA256 (symmetric) or RS256 (asymmetric, ideal for microservices). Access token (short-lived) + Refresh token (HttpOnly cookie, XSS-safe).
👤
Claims — Standard: sub, email, role, exp. Custom for ERP: department_id, tenant_id. Readable in controller via User.FindFirst("department_id")?.Value.
🛡️
Policy-based Auth — Custom requirements + handlers: SameDepartmentRequirement. More flexible than roles for complex ERP business rules. Apply via [Authorize(Policy = "SameDepartment")].
Code Reference

public string GenerateToken(AppUser user) { var claims = new[] { new Claim(ClaimTypes.NameIdentifier, user.Id.ToString()), new Claim(ClaimTypes.Email, user.Email), new Claim(ClaimTypes.Role, user.Role), new Claim("department_id", user.DeptId.ToString()), }; var key = new SymmetricSecurityKey(Encoding.UTF8.GetBytes(_cfg["Jwt:Key"]!)); var token = new JwtSecurityToken( issuer: _cfg["Jwt:Issuer"], audience: _cfg["Jwt:Audience"], claims: claims, expires: DateTime.UtcNow.AddHours(8), signingCredentials: new SigningCredentials(key, SecurityAlgorithms.HmacSha256)); return new JwtSecurityTokenHandler().WriteToken(token); }

Overview of Blazor (.NET 8)
ServerWASMUnified RenderingSSR
Blazor builds interactive web UIs in C# instead of JavaScript. .NET 8 introduced the Unified Rendering Model — a single app can mix Server-Side Rendering (SSR for non-interactive pages), Blazor Server (SignalR), and Blazor WASM per component. This eliminates the old binary Server-vs-WASM choice and lets you choose the right mode for each component.
Topics Covered
🖥️
Blazor Server — UI runs server-side; browser sends events via SignalR and receives DOM diffs. Fast initial load, full .NET access. Latency-sensitive for real-time interactions at scale.
🌐
Blazor WASM — .NET runtime runs in the browser via WebAssembly. True SPA, offline-capable. Larger initial bundle; limited to browser sandbox APIs.
🔗
Unified Rendering (.NET 8) — Per-component: @rendermode InteractiveServer, InteractiveWebAssembly, InteractiveAuto. SSR (no rendermode) for static pages — fastest perceived load, SEO-friendly.

As Blazor apps grow, state management becomes the central design challenge. The Virtualize component renders only visible rows — essential for a student list with thousands of records. The component lifecycle governs when to fetch data and when to dispose resources — failure to unsubscribe events causes memory leaks in long-running Blazor Server sessions.
Topics Covered

AppState Pattern — Scoped service as a shared store. Exposes event Action? OnChange. Components subscribe in OnInitialized, call StateHasChanged() on notification, and must unsubscribe in Dispose().

Clean Architecture in .NET
Domain LayerCQRSMediatRTestability
Clean Architecture organises code into concentric circles where inner layers never reference outer layers. Domain entities know nothing about EF Core. Application use cases know nothing about ASP.NET. Swap SQL Server for Cosmos DB, or REST for gRPC — without touching business logic. Every layer is independently testable.
Topics Covered
🎯
Domain Layer — Entities, Value Objects, Domain Events, Aggregate Roots, Repository Interfaces. Zero external dependencies — pure C#.
⚙️
Application Layer — CQRS + MediatR — Commands (writes: EnrollStudentCommand) and Queries (reads: GetStudentByIdQuery) handled by IRequestHandler<TRequest, TResponse>. Pipeline behaviours: logging, validation, transactions.
🗄️
Infrastructure Layer — EF Core, repository implementations, email, Azure Blob, Azure OpenAI adapters. All implement Application-layer interfaces.
🌐
Presentation Layer — Thin controllers or Blazor pages. Maps HTTP → MediatR. Zero business logic. Completely swappable.

Azure OpenAIGPT-4oTokensAI Patterns
Azure AI Services provides enterprise-grade AI — with data privacy, private networking, compliance, and SLA guarantees the public OpenAI API does not offer. Your data never trains OpenAI's models.
