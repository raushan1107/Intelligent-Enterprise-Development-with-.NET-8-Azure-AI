Classes, Objects, Constructors & Encapsulation

In this lab you will transition from writing procedural code into object-oriented programming using C#. You will learn how to create classes, build objects, initialize data using constructors, expose data using properties, and protect data using encapsulation.

📦 Modules covered: 6, 7, 8, 9 ⏱ Duration: ~90 minutes 🛠 Tool: Visual Studio 2022 🎯 Builds on: Lab 2
MODULE 6

Classes & Objects

1 Why do we need classes?

As applications grow, storing everything in separate variables becomes difficult to manage. Related data should stay together.

Without classes
string studentName = "Raushan";
int age = 28;
string course = "AZ-204";
string city = "Noida";
With classes
Student student = new Student();

student.Name = "Raushan";
student.Age = 28;
student.Course = "AZ-204";
Think of a class as a blueprint. A blueprint describes what an object should contain. An object is the actual thing created from that blueprint.
2 Creating your first class

A class groups related data and behavior together.

Creating a Student class 
class Student
{
    public string Name;
    public int Age;
    public string Course;
}
public means the field can be accessed from outside the class. You will later learn how to protect fields using encapsulation.
3 Creating objects from a class

Once the class exists, you can create multiple objects from it. Each object stores its own data.

Creating Student objects 
Student s1 = new Student();

s1.Name = "Raushan";
s1.Age = 28;
s1.Course = "AZ-204";

Student s2 = new Student();

s2.Name = "Ritu";
s2.Age = 22;
s2.Course = "Blazor";

Console.WriteLine(s1.Name);
Console.WriteLine(s2.Name);
Output
Raushan
Ritu

  • Create two Product objects.
  • Assign different values to both objects.
  • Print all values using Console.WriteLine().
MODULE 7

Constructors

1 Default constructor

A constructor runs automatically when an object is created. It is commonly used to initialize data.

Default constructor example 
class Student
{
    public string Name;
    public int Age;

    public Student()
    {
        Console.WriteLine("Student object created");
    }
}

Student s1 = new Student();
Output
Student object created
A constructor:
  • Has the same name as the class
  • Does not have a return type
  • Runs automatically during object creation
2 Parameterized constructor

Instead of assigning values one-by-one after object creation, constructors can accept values directly.

Parameterized constructor 
class Student
{
    public string Name;
    public int Age;

    public Student(string name, int age)
    {
        Name = name;
        Age = age;
    }
}

Student s1 = new Student("Raushan", 28);

Console.WriteLine(s1.Name);
Console.WriteLine(s1.Age);
Output
Raushan
28
3 Constructor overloading

Just like methods, constructors can also be overloaded.

Multiple constructors 
class Student
{
    public string Name;
    public int Age;

    public Student()
    {
    }

    public Student(string name)
    {
        Name = name;
    }

    public Student(string name, int age)
    {
        Name = name;
        Age = age;
    }
}

Student s1 = new Student();
Student s2 = new Student("Raushan");
Student s3 = new Student("Ritu", 22);
✦ Try It
  • Create an Employee class.
  • Create 2 constructors:
    • One accepting only name
    • One accepting name and salary
  • Create objects using both constructors.
MODULE 8

Properties

1 Why properties are needed

Public fields allow anyone to directly change values, even invalid ones.

Problem with public fields 
Student s1 = new Student();

s1.Age = -500;
This creates invalid data. Properties help us control how values are accessed and changed.
2 Auto-properties

Properties are the modern and recommended way to expose data in C#.

Using auto-properties 
class Student
{
    public string Name { get; set; }

    public int Age { get; set; }
}

Student s1 = new Student();

s1.Name = "Raushan";
s1.Age = 28;

Console.WriteLine(s1.Name);
Console.WriteLine(s1.Age);
  • get → read the value
  • set → assign the value
3 Read-only properties

Some values should not change after object creation.

Read-only property 
class Student
{
    public int StudentId { get; }

    public Student(int id)
    {
        StudentId = id;
    }
}

Student s1 = new Student(101);

Console.WriteLine(s1.StudentId);
Because there is no set, the value cannot be modified outside the constructor.
MODULE 9

Encapsulation

1 Private fields

Encapsulation means hiding internal data and controlling access through methods or properties.

Using private fields 
class Student
{
    private int _age;

    public int Age
    {
        get { return _age; }

        set
        {
            if (value >= 0)
            {
                _age = value;
            }
        }
    }
}
Now invalid values are rejected automatically.
2 Validation inside properties

Properties are a perfect place for validation logic.

Validation example 
class Employee
{
    private decimal _salary;

    public decimal Salary
    {
        get { return _salary; }

        set
        {
            if (value > 0)
            {
                _salary = value;
            }
            else
            {
                Console.WriteLine("Invalid salary");
            }
        }
    }
}
✦ Try It
  • Create validation for marks.
  • Marks should only allow values between 0 and 100.
  • Print an error message if invalid data is assigned.
3 Controlled methods instead of direct modification

Objects should protect their own data. Instead of directly changing balance values, create methods that safely manage changes.

Bank account example 
class BankAccount
{
    private decimal _balance;

    public void Deposit(decimal amount)
    {
        if(amount > 0)
        {
            _balance += amount;
        }
    }

    public void Withdraw(decimal amount)
    {
        if(amount > 0 && amount <= _balance)
        {
            _balance -= amount;
        }
    }

    public void PrintBalance()
    {
        Console.WriteLine(_balance);
    }
}
This protects the object from invalid operations like negative withdrawals or overdrafts.
