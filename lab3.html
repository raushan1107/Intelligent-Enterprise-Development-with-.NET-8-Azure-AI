MODULE 6

Classes & Objects

Imagine handling 500 students without grouping related data—it becomes messy and repetitive[cite: 60, 62, 63]. Classes solve this by providing a blueprint for data[cite: 66, 83].

1 Create Your First Class
Student.cs blueprint [cite: 73]
class Student
{
    public string Name;
    public int Age;
    public string Course;
}
💡

Analogy: A class is like a house blueprint. It describes rooms and windows, but it isn't a real house until you construct an object from it[cite: 96, 101, 102].

2 Instantiate Objects

Use the new keyword to create objects in memory[cite: 117, 118].

Creating s1 and s2 [cite: 109, 134]
Student s1 = new Student();
s1.Name = "Raushan";
s1.Age = 28;

Student s2 = new Student();
s2.Name = "Ritu"; // Each object stores its own data [cite: 141]
✦ Try It
  • Create a Product class with Name, Price, and Quantity[cite: 149].
  • Instantiate two products and print their values[cite: 154, 156].
MODULE 7

Constructors

Constructors ensure objects are properly initialized, preventing developers from forgetting to assign values[cite: 165].

1 Default Constructor
Runs automatically on 'new' [cite: 176, 190]
public Student() 
{
    Console.WriteLine("Student object created");
}
2 Parameterized Constructor

Pass values directly during creation[cite: 192].

Assigning values via constructor [cite: 201]
public Student(string name, int age) 
{
    Name = name;
    Age = age;
}
3 Overloading

Just like methods, you can have multiple constructors with different parameters[cite: 225, 226].

Multiple entry points [cite: 249, 250]
var s1 = new Student();
var s2 = new Student("Raushan");
var s3 = new Student("Ritu", 22);
MODULE 8

Properties

Public fields allow invalid data (e.g., Age = -500)[cite: 262, 266]. Properties provide control using get and set[cite: 269, 279].

1 Auto-Properties
Shorthand syntax [cite: 276]
public string Name { get; set; }
public int Age { get; set; }
2 Read-Only Properties

Remove the set to prevent changes after initialization[cite: 302, 313].

Id cannot be changed later [cite: 302]
public int StudentId { get; }
MODULE 9

Encapsulation

Encapsulation hides internal data and controls access so the object protects itself[cite: 324, 327, 369].

1 Validation in Setters
Protecting the _age field [cite: 334, 344]
private int _age;
public int Age 
{
    get => _age;
    set { if (value >= 0) _age = value; }
}
2 Safe Operations (Bank Example)

Instead of direct balance changes, use methods to ensure logic (no negative withdrawals)[cite: 371, 373].

BankAccount logic [cite: 387, 389]
public void Withdraw(decimal amount) 
{
    if(amount > 0 && amount <= _balance) 
    {
        _balance -= amount;
    }
}
