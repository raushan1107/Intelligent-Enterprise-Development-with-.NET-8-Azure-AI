MODULE 1

Writing Applications in C# and .NET 8

1 What is .NET 8?

.NET 8 is a free, cross-platform framework by Microsoft. You write code in C# and .NET handles running it on Windows, Linux, or Mac.

Think of it in layers: your C# code sits on top, the Base Class Library (BCL) gives you built-in tools like Console, Math, File, and the runtime (CLR) executes everything.

Concept
// The .NET stack — from your code to the CPU

Your C# Code
    ↓
Base Class Library  (String, List, File, Math…)
    ↓
CLR — Common Language Runtime
    ↓
Operating System (Windows / Linux / Mac)
    ↓
CPU
2 How the CLR runs your code

When you press F5, three things happen very quickly:

Compilation pipeline
Program.cs  (your C# source)
    ↓  Roslyn Compiler  (dotnet build)
Program.dll  (IL — Intermediate Language, CPU-neutral)
    ↓  CLR at runtime
    ├─ Loads your types and methods
    ├─ JIT compiles IL → native machine code
    ├─ Garbage Collector manages memory for you
    └─ Runs on your CPU
💡

IL is CPU-neutral. That's why the same .dll file runs on Windows, Linux, and Mac without recompiling. The CLR's JIT compiler converts it to the right native code on each machine.

MODULE 2

Types of Data and Expressions

1 Value Types vs Reference Types

Every piece of data in C# has a type. The type decides what values it can hold and how it behaves when you copy it.

Value types — copy gives you an independent copy 
int a = 10;
int b = a;     // b gets a COPY of a's value
b = 99;        // changing b does NOT affect a

Console.WriteLine(a);  // 10  (unchanged)
Console.WriteLine(b);  // 99
Reference types — two variables can share the same object 
int[] arr1 = { 1, 2, 3 };
int[] arr2 = arr1;   // arr2 points to the SAME array
arr2[0] = 999;      // modifying arr2 also changes arr1!

Console.WriteLine(arr1[0]);  // 999 — same object
ℹ

Common value types: int, double, float, decimal, bool, char, byte, long
Common reference types: string, class, array, List<T>, object

✦ Try It
  • Declare an int, copy it, change the copy — confirm the original is unchanged.
  • Try the same with string — notice strings behave differently from arrays even though both are reference types. Why? (Hint: strings are immutable.)
2 Variables and Constants
Variables, var, and const 
// Explicit type — you name the type yourself
int    age        = 25;
double price      = 99.99;
bool   isEnrolled = true;
string name       = "Raushan";

// var — compiler figures out the type automatically
var course = "Azure AI";   // compiler knows: string
var days   = 11;           // compiler knows: int

// const — value is fixed, cannot be changed
const double Pi = 3.14159;
// Pi = 3.0;  ← this would be a compile error

Console.WriteLine(name + " — " + course + " — Day " + days);
Output
Raushan — Azure AI — Day 11

3 Operators
Arithmetic, comparison, logical 
int a = 10, b = 3;

// Arithmetic
Console.WriteLine(a + b);   // 13
Console.WriteLine(a - b);   // 7
Console.WriteLine(a * b);   // 30
Console.WriteLine(a / b);   // 3  (integer division — remainder dropped)
Console.WriteLine(a % b);   // 1  (remainder / modulus)

// Compound assignment — shorthand
a += 5;   // same as: a = a + 5
a++;       // same as: a = a + 1

// Comparison — result is true or false
Console.WriteLine(a == b);   // false
Console.WriteLine(a > b);    // true
Console.WriteLine(a != b);   // true

// Logical
bool hasAccount = true;
bool hasPaid    = false;
Console.WriteLine(hasAccount && hasPaid);  // false — both must be true
Console.WriteLine(hasAccount || hasPaid);  // true  — at least one true
Console.WriteLine(!hasAccount);           // false — flips the value
✦ Try It

  • Use % to check if a number is even or odd.
  • Write a condition: is a student eligible if age ≥ 18 AND is enrolled?
4 Type Conversion
Implicit, explicit cast, Parse, TryParse 
// Implicit — safe, no data loss, compiler does it automatically
int    myInt    = 100;
double myDouble = myInt;   // int fits into double safely → 100.0

// Explicit cast — you force it, possible data loss
double pi      = 3.14159;
int    piAsInt = (int)pi;   // decimal part is CUT OFF → 3 (not rounded)

// Parse — convert string to number
string input  = "42";
int    parsed = int.Parse(input);   // works fine → 42

// TryParse — safe version, does NOT crash on bad input
bool ok = int.TryParse("hello", out int result);
Console.WriteLine(ok);      // false
Console.WriteLine(result);  // 0 (default)
⚠

Prefer TryParse over Parse whenever the input comes from a user or external source. int.Parse("abc") will crash your program — TryParse returns false instead.

✦ Try It
  • Cast 3.9 to int — is the result 3 or 4? Confirm your understanding.
  • Use int.Parse("abc") and see what exception appears. Then switch to TryParse to handle it gracefully.
5 Nullable Types
Nullable value types with ? 
// int cannot normally be null — but int? can
int? score = null;   // no value yet
Console.WriteLine(score);   // (blank)

score = 85;
Console.WriteLine(score);   // 85

// ?? gives a fallback value when the variable is null
int? bonus = null;
int finalBonus = bonus ?? 0;   // use 0 if bonus is null
Console.WriteLine(finalBonus);  // 0

MODULE 3

C# Language Constructs

1 Conditions — if, else if, switch
if / else if / else 
int marks = 72;

if (marks >= 90)
{
    Console.WriteLine("Grade: A");
}
else if (marks >= 75)
{
    Console.WriteLine("Grade: B");
}
else if (marks >= 50)
{
    Console.WriteLine("Grade: C");   // ← this runs
}
else
{
    Console.WriteLine("Grade: F");
}
switch — cleaner for fixed choices 
string role = "Trainer";

switch (role)
{
    case "Admin":
        Console.WriteLine("Full access");
        break;
    case "Trainer":
        Console.WriteLine("Can manage courses");   // ← runs
        break;
    case "Student":
        Console.WriteLine("Can view courses");
        break;
    default:
        Console.WriteLine("No access");
        break;
}
✦ Try It
  • Change marks to 45, 75, 95 and run each time — confirm the correct grade prints.
  • Add a grade D for marks between 40 and 49.
  • Rewrite the grade logic using a switch on the letter grade instead of marks.
2 Loops
for — when you know the count 
for (int i = 1; i <= 5; i++)
{
    Console.WriteLine("Day " + i);
}
foreach — for every item in a collection 
string[] courses = { "C#", "Azure AI", "Blazor" };

foreach (string course in courses)
{
    Console.WriteLine(course);
}
while and do-while 
// while — checks condition BEFORE running
int attempts = 0;
while (attempts < 3)
{
    attempts++;
    Console.WriteLine("Attempt " + attempts);
}

// do-while — runs at LEAST ONCE, checks condition AFTER
int count = 0;
do
{
    Console.WriteLine("Processing " + (count + 1));
    count++;
} while (count < 2);

3 Exception Handling

An exception is a runtime error. Without handling it, your program crashes. try/catch lets you deal with it gracefully.

try / catch / finally 
// Without try/catch — this crashes the program
// int result = 10 / 0;  ← DivideByZeroException!

// With try/catch — handles the error instead of crashing
try
{
    int result = 10 / 0;
    Console.WriteLine(result);
}
catch (DivideByZeroException ex)
{
    Console.WriteLine("Error: " + ex.Message);
}
finally
{
    // finally ALWAYS runs — good for cleanup
    Console.WriteLine("Done.");
}
Output
Error: Attempted to divide by zero.
Done.
💡

Use finally for any cleanup that must happen regardless — closing a file, releasing a database connection, logging that an operation finished.

✦ Try It
  • Wrap int.Parse("abc") in a try/catch — catch the FormatException and print a helpful message.
  • Add a second catch block for Exception (the base type) below the specific one. Which one runs?
  • Remove the try/catch entirely and run the program — observe what the unhandled exception looks like in the console.
