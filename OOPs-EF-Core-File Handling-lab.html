Create a base class Person with Name and Email. Derive Trainer (adds Expertise) and Learner (adds EnrolledCourse). Each should override Introduce() using base.Introduce() first.

Inheritance.cs — starter
class Person
{
    public string Name  { get; set; }
    public string Email { get; set; }

    public Person(string name, string email)
    { Name = name; Email = email; }

    public virtual void Introduce()
        => Console.WriteLine($"Hi, I am {Name} ({Email})");
}

class Trainer : Person
{
    public string Expertise { get; set; }

    public Trainer(string name, string email, string expertise)
        : base(name, email)          // calls Person constructor
    { Expertise = expertise; }

    public override void Introduce()
    {
        base.Introduce();             // prints Person's line first
        Console.WriteLine($"  Role: Trainer | Expertise: {Expertise}");
    }
}

Interface Contract + Blazor DI Pattern
▼

Define INotificationService with a Send() method. Build EmailNotifier and SmsNotifier. Then swap implementations — your calling code should NOT change at all.

Interfaces.cs — key pattern
// Step 1 — define the contract
interface INotificationService
{
    void Send(string recipient, string message);
}

// Step 2 — implement it two ways
class EmailNotifier : INotificationService
{
    public void Send(string recipient, string message)
        => Console.WriteLine($"[EMAIL] To: {recipient} | {message}");
}

// TODO: Add SmsNotifier : INotificationService

// Step 3 — program to the INTERFACE, not the class
INotificationService notifier = new EmailNotifier();
notifier.Send("raushan@rrskillverse.com", "Welcome!");

// Swap — zero change to calling code
notifier = new SmsNotifier();
notifier.Send("+91-9876543210", "Welcome!");
ℹ

This is the Blazor DI pattern. In Blazor, Program.cs has: builder.Services.AddScoped<INotificationService, EmailNotifier>(); — your component injects the interface and never knows which class it gets.

Abstract Class — Template Method Pattern
▼

Create BaseReport as abstract. It has GenerateHeader() and GenerateFooter() implemented, and an abstract void RenderBody(). The Export() method calls all three in sequence — subclasses only fill in the body.

AbstractClasses.cs — template
abstract class BaseReport
{
    public string Title { get; set; }
    protected BaseReport(string title) { Title = title; }

    protected void GenerateHeader()
        => Console.WriteLine($"===== REPORT: {Title} =====");

    protected void GenerateFooter()
        => Console.WriteLine("--- End of Report ---");

    // abstract = no body. Every subclass MUST override this.
    public abstract void RenderBody();

    // Template Method — controls the sequence
    public void Export()
    {
        GenerateHeader();
        RenderBody();   // calls the subclass version
        GenerateFooter();
    }
}

Build a List<BaseNotification> containing email, SMS, and push objects. Loop through with one foreach and call Dispatch() — each type delivers differently without an if/else.

Polymorphism.cs — dispatcher
List<BaseNotification> queue = new()
{
    new EmailNotification("raushan@rrskillverse.com", "Invoice ready"),
    new SmsNotification("+91-9876543210", "OTP: 482910"),
    new PushNotification("device-abc", "New lesson unlocked!"),
    new EmailNotification("admin@koenig.com", "Server alert"),
};

// One loop — each object knows how to handle itself
foreach (BaseNotification n in queue)
{
    n.Dispatch();
}

// Type checking at runtime
foreach (BaseNotification n in queue)
{
    if (n is EmailNotification email)
        Console.WriteLine($"  [AUDIT] Email to: {email.Address}");
}

Static vs Instance Members
▼

Build an Enrollment class where each object has its own StudentName and CourseName (instance), but TotalEnrollments is shared across ALL objects (static).

StaticVsInstance.cs
class Enrollment
{
    // INSTANCE — each object has its own copy
    public string StudentName { get; set; }
    public string CourseName  { get; set; }

    // STATIC — one copy, shared across ALL Enrollment objects
    public static int TotalEnrollments { get; private set; } = 0;

    public Enrollment(string name, string course)
    {
        StudentName = name;
        CourseName  = course;
        TotalEnrollments++;   // increments the shared counter
    }
}

// Usage
var e1 = new Enrollment("Raushan", "AZ-204");
var e2 = new Enrollment("Rahul",   "PL-300");
var e3 = new Enrollment("Priya",   "AZ-204");

Console.WriteLine(Enrollment.TotalEnrollments); // 3

Build a payment processor that throws a custom EnrollmentException when the amount is invalid or payment is unpaid. Catch specific exceptions before the general one.

ExceptionHandling.cs
// Custom exception — inherit from Exception
class EnrollmentException : Exception
{
    public string ErrorCode { get; }
    public EnrollmentException(string message, string code)
        : base(message) { ErrorCode = code; }
}

// Throwing and catching
static void EnrollStudent(string name, bool isPaid)
{
    if (string.IsNullOrWhiteSpace(name))
        throw new ArgumentException("Name required.");

    if (!isPaid)
        throw new EnrollmentException(
            $"Payment not confirmed for {name}", "ENR_001");

    Console.WriteLine($"{name} enrolled successfully.");
}

try { EnrollStudent("Rahul", isPaid: false); }
catch (EnrollmentException ex)
    { Console.WriteLine($"Blocked [{ex.ErrorCode}]: {ex.Message}"); }
catch (ArgumentException ex)
    { Console.WriteLine($"Validation: {ex.Message}"); }
finally
    { Console.WriteLine("Enrollment attempt logged."); }

Create Employee Entity + AppDbContext
▼

Add two files to EFCoreDemo: an Employee entity class with data annotations, and an AppDbContext with the LocalDB connection string.

Employee.cs
using System.ComponentModel.DataAnnotations;
using System.ComponentModel.DataAnnotations.Schema;

namespace EFCoreDemo;

public class Employee
{
    public int     Id         { get; set; }   // PK — auto int IDENTITY

    [Required][MaxLength(100)]
    public string  Name       { get; set; } = "";

    [Required][MaxLength(150)][EmailAddress]
    public string  Email      { get; set; } = "";

    [Required][MaxLength(50)]
    public string  Department { get; set; } = "";

    [Column(TypeName = "decimal(18,2)")]
    public decimal Salary     { get; set; }

    public DateTime JoiningDate { get; set; }
    public bool     IsActive    { get; set; } = true;

    public override string ToString() =>
        $"[{Id}] {Name,-20} | {Department,-12} | ₹{Salary,8} | Active:{IsActive}";
}
AppDbContext.cs
using Microsoft.EntityFrameworkCore;

namespace EFCoreDemo;

public class AppDbContext : DbContext
{
    // DbSet = represents the Employees TABLE
    public DbSet<Employee> Employees { get; set; }

    protected override void OnConfiguring(DbContextOptionsBuilder options)
    {
        options.UseSqlServer(
            @"Server=(localdb)\MSSQLLocalDB;Database=EFCoreDemoDB;" +
            "Trusted_Connection=True;TrustServerCertificate=True;"
        );
    }
    // No seeding needed — we add data in Program.cs
}

Full CRUD in Program.cs
▼

Open Program.cs and walk through Add → Read All → Update → Soft Delete → Hard Delete → Final Read, using SaveChanges() after every write operation.

Program.cs — complete CRUD demo
using EFCoreDemo;

using AppDbContext context = new AppDbContext();

// ── ADD ───────────────────────────────────────────────────
Employee emp = new Employee
{
    Name = "Rahul Gupta", Email = "rahul@example.com",
    Department = "HR", Salary = 60000m, JoiningDate = DateTime.Today
};
context.Employees.Add(emp);
context.SaveChanges();            // INSERT INTO Employees ...
Console.WriteLine("Added — Id=" + emp.Id);

// ── READ ALL ──────────────────────────────────────────────
Console.WriteLine("\nAll Employees:");
foreach (Employee e in context.Employees)
    Console.WriteLine("  " + e);

// ── UPDATE ────────────────────────────────────────────────
Employee? toUpdate = context.Employees
    .FirstOrDefault(e => e.Name == "Rahul Gupta");
if (toUpdate != null)
{
    toUpdate.Salary     = 75000m;
    toUpdate.Department = "Operations";
    context.SaveChanges();         // UPDATE Employees SET ...
    Console.WriteLine("\nUpdated: " + toUpdate);
}

// ── SOFT DELETE — mark inactive ───────────────────────────
if (toUpdate != null)
{
    toUpdate.IsActive = false;
    context.SaveChanges();         // UPDATE SET IsActive=0
}

// ── READ ACTIVE ONLY ──────────────────────────────────────
Console.WriteLine("\nActive Employees:");
foreach (Employee e in context.Employees.Where(e => e.IsActive))
    Console.WriteLine("  " + e);

// ── HARD DELETE ───────────────────────────────────────────
Employee? toDelete = context.Employees.Find(emp.Id);
if (toDelete != null)
{
    context.Employees.Remove(toDelete);
    context.SaveChanges();         // DELETE FROM Employees WHERE Id=...
    Console.WriteLine($"\nDeleted Id={emp.Id}");
}

// ── FINAL STATE ───────────────────────────────────────────
Console.WriteLine("\nFinal state:");
foreach (Employee e in context.Employees)
    Console.WriteLine("  " + e);

Exercise 12
Create, Write, Append, Delete
▼

Use File.WriteAllText, File.AppendAllText, and File.Delete — the simple one-shot API. Always use Path.Combine and check File.Exists before reading or deleting.

FileHandling.cs — Steps 1–4
string dir  = Path.Combine(AppDomain.CurrentDomain.BaseDirectory, "demo-files");
Directory.CreateDirectory(dir);

// STEP 1 — CREATE / OVERWRITE
string path = Path.Combine(dir, "notes.txt");
File.WriteAllText(path, "Hello from RR Skillverse!\nC# file handling is easy.");
Console.WriteLine("Created: " + path);

// STEP 2 — READ BACK
if (File.Exists(path))
    Console.WriteLine(File.ReadAllText(path));

// STEP 3 — APPEND (does NOT erase existing content)
File.AppendAllText(path, "\nAppended line — original is safe.");
Console.WriteLine("\nAfter append:");
Console.WriteLine(File.ReadAllText(path));

// STEP 4 — SAFE DELETE
if (File.Exists(path))
{
    File.Delete(path);
    Console.WriteLine("Deleted: " + File.Exists(path));   // False
}
⚠

WriteAllText overwrites. AppendAllText does not. If you need to update a specific line — ReadAllLines → modify the array → WriteAllLines back.

Exercise 13
ReadAllLines vs StreamReader — When to Use Each
▼

File.ReadAllLines loads the whole file into memory. StreamReader reads one line at a time — essential for large files or when you need to stop early.

StreamReader — read + filter + stop early
string csvPath = Path.Combine(dir, "courses.csv");

// Read ALL lines
using (StreamReader reader = new StreamReader(csvPath))
{
    string? line;
    while ((line = reader.ReadLine()) != null)  // null = end of file
        Console.WriteLine(line);
}

// Filter while reading — no need to load everything
using (StreamReader reader = new StreamReader(csvPath))
{
    reader.ReadLine();  // skip header
    string? line;
    while ((line = reader.ReadLine()) != null)
        if (line.Contains("Advanced")) Console.WriteLine("→ " + line);
}

// Stop early — find first expensive course
using (StreamReader reader = new StreamReader(csvPath))
{
    reader.ReadLine();
    string? line;
    while ((line = reader.ReadLine()) != null)
    {
        string[] parts = line.Split(',');
        if (parts.Length >= 4 && int.TryParse(parts[3], out int price) && price > 5000)
        {
            Console.WriteLine("First expensive: " + line);
            break;  // stop — don't read the rest
        }
    }
}

Exercise 14
StreamWriter — Progress Bar + Real App Demo
▼

Use StreamWriter to write a CSV row by row with a live progress indicator. Always wrap in a using block — it guarantees the file closes even if an exception occurs.

StreamWriter — CSV with progress
string csvPath = Path.Combine(dir, "courses.csv");

string[] rows =
{
    "Id,Title,Level,Price",
    "1,AZ-204,Intermediate,6000",
    "2,PL-300,Beginner,3000",
    "3,.NET 8 + Azure,Advanced,8000",
    "4,GitHub Copilot,Beginner,2500",
    "5,MS-700,Intermediate,4000"
};

// 'append: false' = create or overwrite
using (StreamWriter writer = new StreamWriter(csvPath, append: false))
{
    for (int i = 0; i < rows.Length; i++)
    {
        writer.WriteLine(rows[i]);

        // Live progress indicator
        int pct = (int)((i + 1) / (double)rows.Length * 100);
        Console.Write($"\r  Writing... {pct}%   ");
        System.Threading.Thread.Sleep(80);
    }
}   // ← file CLOSED and FLUSHED here automatically

Console.WriteLine("\n✓ CSV written.");

// Append 2 more rows without losing existing data
using (StreamWriter writer = new StreamWriter(csvPath, append: true))
{
    writer.WriteLine("6,AZ-900,Beginner,1500");
    writer.WriteLine("7,AI-102,Intermediate,5500");
}
Console.WriteLine("✓ 2 rows appended.");
💡

The using rule: always use using with StreamWriter and StreamReader. Without it, if your code throws an exception mid-write, the file stays locked and nothing else can open it until the process ends.
