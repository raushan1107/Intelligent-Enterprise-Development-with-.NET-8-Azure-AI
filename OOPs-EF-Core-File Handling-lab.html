Exercise 1 — Inheritance
Build a Person → Trainer → Learner chain
▼

🧠 What you will learn

Inheritance lets one class reuse the code of another. Instead of writing Name and Email in both Trainer and Learner, we write them once in Person and both classes get them for free.

The child class uses : ParentClass to inherit. Use base() to call the parent constructor. Use override to change a parent method.

1

Add a new file called Inheritance.cs to your project.
Right-click project → Add → New Item → C# Class → name it Inheritance.cs

2

Replace the entire file content with this code:

Inheritance.cs
namespace DotNetAzureAI;

// BASE CLASS — code written here is shared with ALL child classes
class Person
{
    public string Name  { get; set; } = "";
    public string Email { get; set; } = "";

    public Person(string name, string email)
    {
        Name  = name;
        Email = email;
    }

    // 'virtual' = child classes are ALLOWED to override this
    public virtual void Introduce()
    {
        Console.WriteLine($"Hi, I am {Name} ({Email})");
    }
}

// DERIVED CLASS — inherits everything from Person
// Trainer IS a Person — gets Name, Email, Introduce() for free
class Trainer : Person
{
    public string Expertise { get; set; } = "";

    // base(name, email) calls the Person constructor above
    public Trainer(string name, string email, string expertise)
        : base(name, email)
    {
        Expertise = expertise;
    }

    // override = replace the parent version with our own
    public override void Introduce()
    {
        base.Introduce();   // calls Person.Introduce() first
        Console.WriteLine($"  Role: Trainer | Expertise: {Expertise}");
    }

    public void Teach()
    {
        Console.WriteLine($"{Name} is now teaching {Expertise}");
    }
}

// YOUR TURN — add Learner below this comment
// It should: inherit Person, add EnrolledCourse property,
// override Introduce() to also print the course name

// ----- RUNNER -----
public static class InheritanceDemo
{
    public static void Run()
    {
        Console.WriteLine("=== Exercise 1: Inheritance ===");
        Console.WriteLine();

        Trainer t = new Trainer("Raushan", "raushan@rrskillverse.com", "Azure & .NET");
        t.Introduce();
        t.Teach();

        Console.WriteLine();

        // Uncomment after you add the Learner class:
        // Learner l = new Learner("Rahul", "rahul@example.com", "AZ-204");
        // l.Introduce();

}

Exercise 2 — Interfaces
INotificationService — swap Email for SMS without changing the caller
▼

🧠 What you will learn

An interface is a contract. It says what methods must exist — but not how they work. Any class that signs the contract must provide those methods.

In Blazor, your pages inject an interface. The DI container decides which class to give you. This means you can swap EmailNotifier for SmsNotifier without touching a single page.

1

Add a new file called Interfaces.cs.

2

Paste this complete code:

Interfaces.cs
namespace DotNetAzureAI;

// THE CONTRACT — anyone who signs this MUST provide Send()
interface INotificationService
{
    void Send(string recipient, string message);
}

// IMPLEMENTATION 1 — delivers via email
class EmailNotifier : INotificationService
{
    public void Send(string recipient, string message)
    {
        Console.WriteLine($"[EMAIL] To: {recipient}");
        Console.WriteLine($"        Body: {message}");
    }
}

// IMPLEMENTATION 2 — delivers via SMS
class SmsNotifier : INotificationService
{
    public void Send(string recipient, string message)
    {
        Console.WriteLine($"[SMS]   To: {recipient}");
        Console.WriteLine($"        Text: {message}");
    }
}

// ----- RUNNER -----
public static class InterfacesDemo
{
    public static void Run()
    {
        Console.WriteLine("=== Exercise 2: Interfaces ===");
        Console.WriteLine();

        // Variable type is the INTERFACE — not the class
        // This is how Blazor DI works
        INotificationService notifier = new EmailNotifier();
        notifier.Send("raushan@rrskillverse.com", "Welcome to RR Skillverse!");

        Console.WriteLine();

        // SWAP — only this one line changes. The Send() call below stays identical.
        notifier = new SmsNotifier();
        notifier.Send("+91-9876543210", "Welcome to RR Skillverse!");
    }
}

Exercise 3 — Abstract Classes
BaseReport — shared structure, each format fills in its own body
▼

🧠 What you will learn

Abstract class = halfway between a regular class and an interface. It can have real implemented methods (like GenerateHeader) AND abstract methods with no body that every subclass must implement.

Key rule: you cannot create an object directly from an abstract class. new BaseReport() is a compile error.

1

Add a new file called AbstractClasses.cs and paste this code:

AbstractClasses.cs
namespace DotNetAzureAI;

// ABSTRACT — cannot be instantiated directly
abstract class BaseReport
{
    public string Title { get; set; }
    protected BaseReport(string title) { Title = title; }

    // IMPLEMENTED — shared by all subclasses
    protected void GenerateHeader()
    {
        Console.WriteLine($"==============================");
        Console.WriteLine($"  REPORT: {Title}");
        Console.WriteLine($"==============================");
    }

    protected void GenerateFooter()
        => Console.WriteLine("--- End of Report ---\n");

    // ABSTRACT — no body. Every subclass MUST provide this.
    public abstract void RenderBody();

    // Template Method — controls the sequence. Subclasses fill in the blank.
    public void Export()
    {
        GenerateHeader();
        RenderBody();     // calls the SUBCLASS version
        GenerateFooter();
    }
}

class PdfReport : BaseReport
{
    public PdfReport(string title) : base(title) { }

    public override void RenderBody()
    {
        Console.WriteLine("  [PDF] Embedding fonts and images...");
        Console.WriteLine("  [PDF] Writing binary stream to disk.");
    }
}

// YOUR TURN: Add ExcelReport and HtmlReport below
// Each must override RenderBody() with its own 2 Console.WriteLine lines

public static class AbstractDemo
{
    public static void Run()
    {
        Console.WriteLine("=== Exercise 3: Abstract Classes ===");
        Console.WriteLine();

        // Store as base type — polymorphism in action
        List<BaseReport> reports = new()

            // new ExcelReport("Trainer Utilisation"),   // uncomment after adding
            // new HtmlReport("Course Dashboard"),       // uncomment after adding
        };

        foreach (BaseReport r in reports)
            r.Export();
    }
}

Exercise 4 — Polymorphism
One loop, four notification types — each dispatches itself
▼

🧠 What you will learn

Polymorphism = same method call, different behaviour. A list of mixed types, one loop — each object does the right thing automatically. No if/else needed.

Also covers method overloading (same method name, different parameters — compiler picks the right one at build time).

1

Add Polymorphism.cs and paste:

Polymorphism.cs
namespace DotNetAzureAI;

abstract class AppNotification
{
    public string Message { get; set; } = "";
    protected AppNotification(string msg) { Message = msg; }
    public abstract void Dispatch();
}

class AppEmail : AppNotification
{
    public string Address { get; } = "";
    public AppEmail(string addr, string msg) : base(msg) { Address = addr; }
    public override void Dispatch()
        => Console.WriteLine($"[EMAIL] → {Address} | \"{Message}\"");
}

class AppSms : AppNotification
{
    public string Phone { get; } = "";
    public AppSms(string phone, string msg) : base(msg) { Phone = phone; }
    public override void Dispatch()
        => Console.WriteLine($"[SMS]   → {Phone} | \"{Message}\"");
}

class AppPush : AppNotification
{
    public string Token { get; } = "";
    public AppPush(string token, string msg) : base(msg) { Token = token; }
    public override void Dispatch()
        => Console.WriteLine($"[PUSH]  → {Token} | \"{Message}\"");
}

// Method Overloading = same name, different parameters
class AppLogger
{
    public void Log(string msg)
        => Console.WriteLine($"[INFO   ] {msg}");

    public void Log(string msg, string source)
        => Console.WriteLine($"[INFO   ] [{source}] {msg}");

    public void Log(int statusCode, string path)
        => Console.WriteLine($"[HTTP   ] {statusCode} on {path}");
}

public static class PolymorphismDemo
{
    public static void Run()
    {
        Console.WriteLine("=== Exercise 4: Polymorphism ===");
        Console.WriteLine();

        // Runtime polymorphism — one loop, different Dispatch() per type
        List<AppNotification> queue = new()
        {
            new AppEmail("raushan@rrskillverse.com", "Invoice ready"),
            new AppSms("+91-9876543210", "OTP: 482910"),
            new AppPush("device-abc-123", "New lesson unlocked!"),
            new AppEmail("admin@koenig.com", "Server alert"),
        };

        Console.WriteLine("Dispatching notifications:");
        foreach (AppNotification n in queue)
            n.Dispatch();

        Console.WriteLine();

        // Compile-time polymorphism — overloading
        Console.WriteLine("Logger overloads:");
        AppLogger logger = new();
        logger.Log("Application started");
        logger.Log("DB connected", "DatabaseService");
        logger.Log(404, "/api/courses/99");

        Console.WriteLine();

        // 'is' = type check at runtime
        Console.WriteLine("Email audit:");
        foreach (AppNotification n in queue)
            if (n is AppEmail email)
                Console.WriteLine($"  Audit → {email.Address}");
    }
}

Exercise 5 — Static vs Instance
Enrollment counter shared across all objects + static utility class
▼

🧠 What you will learn

Instance member: each object has its OWN copy. StudentName is different per enrollment.

Static member: ONE copy, shared across ALL objects and the entire app. TotalEnrollments is the same number regardless of which object you ask.

You already use static every day: Console.WriteLine, Math.Sqrt, DateTime.Now — all static.

1

Add StaticVsInstance.cs and paste:

StaticVsInstance.cs
namespace DotNetAzureAI;

class Enrollment
{
    // INSTANCE — each Enrollment object has its own StudentName and Course
    public string StudentName { get; set; } = "";
    public string Course      { get; set; } = "";

    // STATIC — ONE copy shared across ALL Enrollment objects
    public static int TotalEnrollments { get; private set; } = 0;

    public Enrollment(string name, string course)
    {
        StudentName = name;
        Course      = course;
        TotalEnrollments++;   // every new object increments the shared counter
    }

    public void Print()
        => Console.WriteLine($"  {StudentName} → {Course}  (Total so far: {TotalEnrollments})");
}

// Static class — cannot be instantiated. new AppConfig() = compile error.
static class AppConfig
{
    public static string AppName     { get; } = "RR Skillverse";
    public static string Version     { get; } = "2.1.0";
    public static bool   IsDebugMode { get; set; } = false;

    public static string GetBaseUrl() =>
        IsDebugMode ? "http://localhost:3000" : "https://rrskillverse.com";
}

public static class StaticDemo
{
    public static void Run()
    {
        Console.WriteLine("=== Exercise 5: Static vs Instance ===");
        Console.WriteLine();

        Console.WriteLine("Total before: " + Enrollment.TotalEnrollments);

        Enrollment e1 = new("Raushan", "AZ-204"); e1.Print();
        Enrollment e2 = new("Rahul",   "PL-300"); e2.Print();
        Enrollment e3 = new("Priya",   "AZ-204"); e3.Print();

        // Access static via CLASS NAME — not via object
        Console.WriteLine("Total after:  " + Enrollment.TotalEnrollments);

        Console.WriteLine();
        Console.WriteLine("App: "   + AppConfig.AppName);
        Console.WriteLine("URL: "   + AppConfig.GetBaseUrl());
        AppConfig.IsDebugMode = true;
        Console.WriteLine("URL (debug): " + AppConfig.GetBaseUrl());
    }
}

Exercise 6 — Exception Handling
Custom EnrollmentException — catch specific before general
▼

🧠 What you will learn

Exception handling stops your program from crashing. try = risky code. catch = what to do if it fails. finally = always runs (use for cleanup).

Always catch the most specific exception first. If you put catch(Exception) first, it swallows everything and you never reach the specific ones.

1

Add ExceptionHandling.cs and paste:

ExceptionHandling.cs
namespace DotNetAzureAI;

// Custom exception — inherit from Exception, add your own property
class EnrollmentException : Exception
{
    public string ErrorCode { get; }

    public EnrollmentException(string message, string code)
        : base(message)
    {
        ErrorCode = code;
    }
}

public static class ExceptionDemo
{
    // This method validates and throws custom exceptions
    static void EnrollStudent(string name, bool isPaid)
    {
        if (string.IsNullOrWhiteSpace(name))
            throw new ArgumentException("Student name cannot be empty.");

        if (!isPaid)
            throw new EnrollmentException(
                $"Payment not confirmed for {name}", "ENR_001");

        Console.WriteLine($"  ✓ {name} enrolled successfully.");
    }

    static void TryEnroll(string name, bool isPaid)
    {
        Console.Write($"Enrolling \"{(name == "" ? "" : name)}\"... ");
        try
        {
            EnrollStudent(name, isPaid);
        }
        catch (EnrollmentException ex)   // specific first
        {
            Console.WriteLine($"Blocked [{ex.ErrorCode}]: {ex.Message}");
        }
        catch (ArgumentException ex)      // specific second
        {
            Console.WriteLine($"Validation: {ex.Message}");
        }
        finally
        {
            Console.WriteLine("  → Attempt logged."); // always runs
        }
    }

    public static void Run()
    {
        Console.WriteLine("=== Exercise 6: Exception Handling ===");
        Console.WriteLine();

        TryEnroll("Raushan", isPaid: true);   Console.WriteLine();
        TryEnroll("Rahul",   isPaid: false);  Console.WriteLine();
        TryEnroll("",        isPaid: true);   Console.WriteLine();
    }
}

Exercise 7 — Delegates & LINQ
Action, Func, Predicate — filter and transform a course list
▼

🧠 What you will learn

A delegate is a variable that holds a method reference. Instead of calling a method directly, you store it and pass it around.

Action<T> = holds a void method. Func<T, TResult> = holds a method that returns a value. The => arrow syntax (x => x + 1) creates an anonymous method called a lambda.

LINQ methods like .Where(), .Select(), .OrderBy() all accept lambdas as parameters.

1

Add DelegatesAndLambdas.cs and paste:

DelegatesAndLambdas.cs
namespace DotNetAzureAI;

class CourseItem
{
    public string  Name  { get; set; } = "";
    public string  Level { get; set; } = "";
    public decimal Price { get; set; }
    public override string ToString() => $"  {Name,-28} {Level,-15} ₹{Price}";
}

public static class DelegatesDemo
{
    public static void Run()
    {
        Console.WriteLine("=== Exercise 7: Delegates & LINQ ===");
        Console.WriteLine();

        // Action — holds a void method (no return value)
        Action<string> printLine = msg => Console.WriteLine("  >> " + msg);
        printLine("Hello from a delegate!");
        printLine("Lambdas are just short methods.");

        Console.WriteLine();

        // Func — holds a method that RETURNS a value
        Func<decimal, decimal, decimal> addGst =
            (amount, rate) => amount + (amount * rate / 100);
        Console.WriteLine($"  ₹1000 + 18% GST = ₹{addGst(1000m, 18m)}");

        Console.WriteLine();

        // Predicate — holds a condition check (returns bool)
        Predicate<string> isEmail = e => e.Contains('@') && e.Contains('.');
        Console.WriteLine("  raushan@rrskillverse.com valid? " + isEmail("raushan@rrskillverse.com"));
        Console.WriteLine("  notanemail valid?              " + isEmail("notanemail"));

        Console.WriteLine();

        // LINQ — lambdas as filter, transform, sort
        List<CourseItem> courses = new()
        {
            new() { Name="AZ-204",          Level="Intermediate", Price=6000m },
            new() { Name="PL-300",          Level="Beginner",     Price=3000m },
            new() { Name=".NET 8 + Azure",  Level="Advanced",     Price=8000m },
            new() { Name="GitHub Copilot",  Level="Beginner",     Price=2500m },
            new() { Name="MS-700",          Level="Intermediate", Price=4000m },
        };

        Console.WriteLine("Beginner courses:");
        courses.Where(c => c.Level == "Beginner")
               .ToList()
               .ForEach(c => Console.WriteLine(c));

        Console.WriteLine();

        Console.WriteLine("All names (uppercase):");
        courses.Select(c => c.Name.ToUpper())
               .ToList()
               .ForEach(n => Console.WriteLine("  " + n));

        Console.WriteLine();

        Console.WriteLine("Sorted by price (cheapest first):");
        courses.OrderBy(c => c.Price)
               .ToList()
               .ForEach(c => Console.WriteLine(c));
    }
}

Exercise 9 — Entity & DbContext
Create Employee class + AppDbContext with LocalDB connection
▼

🧠 What you will learn

Entity = a C# class that maps to a database table. EF Core reads your class and creates the table automatically.

DbContext = your connection to the database. One DbSet<T> property = one table. OnConfiguring = where the connection string lives.

1

Add a new file called Employee.cs to the EFCoreDemo project:

Employee.cs
using System.ComponentModel.DataAnnotations;
using System.ComponentModel.DataAnnotations.Schema;

namespace EFCoreDemo;

public class Employee
{
    // int Id named 'Id' = EF Core auto-makes this the Primary Key
    // SQL Server assigns the number (1, 2, 3...) automatically
    public int Id { get; set; }

    // [Required]   → NOT NULL column in the database
    // [MaxLength]  → nvarchar(100) instead of nvarchar(max)
    [Required][MaxLength(100)]
    public string Name { get; set; } = "";

    [Required][MaxLength(150)]
    public string Email { get; set; } = "";

    [Required][MaxLength(50)]
    public string Department { get; set; } = "";

    // [Column(TypeName)] → tells EF Core the exact SQL column type
    [Column(TypeName = "decimal(18,2)")]
    public decimal Salary { get; set; }

    public DateTime JoiningDate { get; set; }

    // Soft delete flag — mark inactive instead of physically removing
    public bool IsActive { get; set; } = true;

    public override string ToString() =>
        $"[{Id}] {Name,-20} | {Department,-12} | ₹{Salary,8} | Active:{IsActive}";
}

using EFCoreDemo;

Console.WriteLine();

// 'using' ensures the DB connection is closed cleanly when done
using AppDbContext context = new AppDbContext();

// ── 1. ADD ───────────────────────────────────────────────────────
Console.WriteLine("Step 1 — Adding employees...");

Employee emp1 = new()

    Name        = "Rahul Gupta",
    Email       = "rahul@example.com",
    Department  = "HR",

};
Employee emp2 = new()

    Name        = "Priya Sharma",
    Email       = "priya@example.com",
    Department  = "Development",
    Salary      = 75000m,
    JoiningDate = DateTime.Today
};

context.Employees.Add(emp1);  // queues INSERT — nothing sent to DB yet
context.Employees.Add(emp2);
context.SaveChanges();        // NOW sends the INSERT to SQL Server

Console.WriteLine($"  Added emp1 → Id={emp1.Id} (SQL Server assigned this)");
Console.WriteLine($"  Added emp2 → Id={emp2.Id}");

// ── 2. READ ALL ──────────────────────────────────────────────────
Console.WriteLine();
Console.WriteLine("Step 2 — All employees in database:");

    Console.WriteLine("  " + e);

// ── 3. UPDATE ────────────────────────────────────────────────────
Console.WriteLine();
Console.WriteLine("Step 3 — Updating Rahul's salary and department...");

Employee? rahul = context.Employees
    .FirstOrDefault(e => e.Name == "Rahul Gupta");

if (rahul != null)
{
    rahul.Salary     = 80000m;       // EF Core sees this change
    rahul.Department = "Operations"; // and this one
    context.SaveChanges();           // sends: UPDATE Employees SET Salary=..., Department=...
    Console.WriteLine("  Updated: " + rahul);
}

// ── 4. SOFT DELETE — mark inactive (preferred in real apps) ──────
Console.WriteLine();
Console.WriteLine("Step 4 — Soft deleting Priya (IsActive = false)...");

Employee? priya = context.Employees
    .FirstOrDefault(e => e.Name == "Priya Sharma");

if (priya != null)
{
    priya.IsActive = false;
    context.SaveChanges();   // UPDATE Employees SET IsActive=0 WHERE Id=2
    Console.WriteLine("  Deactivated: " + priya);
}

// ── 5. FILTER READ — active employees only ───────────────────────
Console.WriteLine();
Console.WriteLine("Step 5 — Active employees only:");
foreach (Employee e in context.Employees.Where(e => e.IsActive))
    Console.WriteLine("  " + e);

// ── 6. HARD DELETE — physically remove ──────────────────────────
Console.WriteLine();
Console.WriteLine($"Step 6 — Hard deleting Rahul (Id={emp1.Id})...");

Employee? toDelete = context.Employees.Find(emp1.Id);
if (toDelete != null)
{
    context.Employees.Remove(toDelete);
    context.SaveChanges();   // DELETE FROM Employees WHERE Id=1
    Console.WriteLine($"  Deleted.");
}

// ── 7. FINAL READ ────────────────────────────────────────────────
Console.WriteLine();
Console.WriteLine("Step 7 — Final state of Employees table:");
foreach (Employee e in context.Employees)
    Console.WriteLine("  " + e);

Console.WriteLine();
Console.WriteLine("Done! Check SQL Server Object Explorer → dbo.Employees → View Data");

Exercise 12 — File Basics
Create a file → Write → Read → Append → Update a line → Delete
▼

🧠 What you will learn

File.WriteAllText = creates or overwrites the whole file in one shot.
File.AppendAllText = adds to the end without erasing anything.
File.ReadAllText = reads the whole file as one string.
File.ReadAllLines = reads as a string array — one element per line.
File.Delete = permanently deletes. Always check File.Exists() first.

1

Add FileHandling.cs to DotNetAzureAI and paste:

FileHandling.cs
namespace DotNetAzureAI;

public static class FileHandlingDemo
{
    // All files created in a 'demo-files' folder next to the .exe
    static readonly string Dir =
        Path.Combine(AppDomain.CurrentDomain.BaseDirectory, "demo-files");

    public static void Run()
    {
        Console.WriteLine("=== Exercise 12: File Handling ===");
        Console.WriteLine();

        // Always create the folder first — safe even if it already exists
        Directory.CreateDirectory(Dir);
        Console.WriteLine("Files go to: " + Dir);
        Console.WriteLine();

        // ── STEP 1: CREATE a file ────────────────────────────────────
        Console.WriteLine("Step 1 — Create and write:");
        string notesPath = Path.Combine(Dir, "notes.txt");

        // WriteAllText creates the file if it doesn't exist
        // If it DOES exist — the old content is ERASED
        File.WriteAllText(notesPath,
            "Line 1: Hello from RR Skillverse!\n" +
            "Line 2: C# file handling is simple.\n" +
            "Line 3: Let's go!\n");

        Console.WriteLine("  ✓ File created.");
        Console.WriteLine("  Exists: " + File.Exists(notesPath));
        Console.WriteLine("  Size:   " + new FileInfo(notesPath).Length + " bytes");

        // ── STEP 2: READ the file back ───────────────────────────────
        Console.WriteLine();
        Console.WriteLine("Step 2 — Read back:");
        string allText = File.ReadAllText(notesPath);
        Console.WriteLine(allText);

        // ReadAllLines = one array element per line
        string[] lines = File.ReadAllLines(notesPath);
        Console.WriteLine("Line count: " + lines.Length);

        // ── STEP 3: APPEND — add without erasing ─────────────────────
        Console.WriteLine();
        Console.WriteLine("Step 3 — Append:");
        File.AppendAllText(notesPath, "Line 4: Appended — original lines safe!\n");
        File.AppendAllText(notesPath, "Line 5: Another appended line.\n");
        Console.WriteLine(File.ReadAllText(notesPath));

        // ── STEP 4: UPDATE a specific line ───────────────────────────
        Console.WriteLine("Step 4 — Update Line 3:");
        string[] allLines = File.ReadAllLines(notesPath);   // read into array
        for (int i = 0; i < allLines.Length; i++)
        {
            if (allLines[i].Contains("Line 3"))
                allLines[i] = "Line 3: UPDATED — now says something different.";
        }
        File.WriteAllLines(notesPath, allLines);           // write back
        Console.WriteLine(File.ReadAllText(notesPath));

        // ── STEP 5: SAFE DELETE ──────────────────────────────────────
        Console.WriteLine("Step 5 — Delete:");
        if (File.Exists(notesPath))
        {
            File.Delete(notesPath);
            Console.WriteLine("  ✓ Deleted.");
        }
        Console.WriteLine("  Exists after delete: " + File.Exists(notesPath));
    }
}

Exercise 13 — StreamWriter
Write a CSV row by row with a live progress bar using StreamWriter
▼

🧠 StreamWriter vs File.WriteAllText — when to use which

File.WriteAllText = one shot, whole content at once. Simple. Best for small files.

StreamWriter = line by line. Best for large files, CSV exports, log files — you write one row at a time and can show progress. Always wrap in using so the file closes even if an exception occurs.

1

Add StreamWriterDemo.cs to your project:

StreamWriterDemo.cs
namespace DotNetAzureAI;

public static class StreamWriterDemo
{
    static readonly string Dir =
        Path.Combine(AppDomain.CurrentDomain.BaseDirectory, "demo-files");

    public static void Run()
    {
        Console.WriteLine("=== Exercise 13: StreamWriter ===");
        Console.WriteLine();

        Directory.CreateDirectory(Dir);
        string csvPath = Path.Combine(Dir, "courses.csv");

        string[] rows = {
            "Id,Title,Level,Price",           // header
            "1,AZ-204,Intermediate,6000",
            "2,PL-300,Beginner,3000",
            "3,.NET 8 + Azure,Advanced,8000",
            "4,GitHub Copilot,Beginner,2500",
            "5,MS-700,Intermediate,4000",
            "6,AZ-900,Beginner,1500",
        };

        Console.WriteLine("Writing courses.csv with StreamWriter:");

        // 'append: false' = create or overwrite
        // 'using' = file is CLOSED and SAVED when this block ends
        //           even if an exception is thrown inside
        using (StreamWriter writer = new StreamWriter(csvPath, append: false))
        {
            for (int i = 0; i < rows.Length; i++)
            {
                writer.WriteLine(rows[i]);

                // Live progress bar — \r goes back to the start of the same line
                int pct = (int)((i + 1) / (double)rows.Length * 100);
                Console.Write($"\r  Writing row {i+1}/{rows.Length}... {pct}%   ");
                System.Threading.Thread.Sleep(150);  // slow it down to see progress
            }
        }   // ← StreamWriter closes and flushes HERE automatically

        Console.WriteLine();  // new line after progress bar
        Console.WriteLine("  ✓ CSV written to: " + csvPath);
        Console.WriteLine();

        // APPEND — add 1 more row without erasing existing content
        Console.WriteLine("Appending 1 more row (append: true):");
        using (StreamWriter writer = new StreamWriter(csvPath, append: true))
        {
            writer.WriteLine("7,AI-102,Intermediate,5500");
        }
        Console.WriteLine("  ✓ Row appended.");
        Console.WriteLine();

        // Read back and display the complete file
        Console.WriteLine("Final contents of courses.csv:");
        Console.WriteLine(new string('-', 50));
        foreach (string line in File.ReadAllLines(csvPath))
            Console.WriteLine("  " + line);
        Console.WriteLine(new string('-', 50));
        Console.WriteLine("Open this file in Notepad to verify: " + csvPath);
    }
}

Exercise 14 — StreamReader
Read line by line, filter results, stop early — without loading the whole file
▼

🧠 StreamReader vs File.ReadAllLines — when to use which

File.ReadAllLines = loads the whole file into memory as a string array. Simple. Best for small files.

StreamReader = reads one line at a time. Best for large files where loading everything into memory would be slow. Also lets you stop reading early when you find what you need.

1

Note: This exercise reads the courses.csv created in Exercise 13. Make sure you ran Exercise 13 first.

Add StreamReaderDemo.cs:

StreamReaderDemo.cs
namespace DotNetAzureAI;

public static class StreamReaderDemo
{
    static readonly string CsvPath = Path.Combine(
        AppDomain.CurrentDomain.BaseDirectory, "demo-files", "courses.csv");

    public static void Run()
    {
        Console.WriteLine("=== Exercise 14: StreamReader ===");
        Console.WriteLine();

        if (!File.Exists(CsvPath))
        {
            Console.WriteLine("courses.csv not found. Run Exercise 13 first.");
            return;
        }

        // ── PATTERN 1: Read ALL lines one by one ─────────────────────
        Console.WriteLine("Pattern 1 — Read every line:");
        int lineNum = 0;
        using (StreamReader reader = new StreamReader(CsvPath))
        {
            string? line;
            while ((line = reader.ReadLine()) != null)  // null = end of file
            {
                lineNum++;
                Console.WriteLine($"  [{lineNum,2}] {line}");
            }
        }
        Console.WriteLine($"  Total lines read: {lineNum}");
        Console.WriteLine();

        // ── PATTERN 2: Filter while reading ──────────────────────────
        // Only print Beginner courses — skip header and non-matching rows
        Console.WriteLine("Pattern 2 — Filter (Beginner courses only):");
        using (StreamReader reader = new StreamReader(CsvPath))
        {
            reader.ReadLine();  // skip the header row
            string? line;
            while ((line = reader.ReadLine()) != null)
            {
                if (line.Contains("Beginner"))
                    Console.WriteLine("  → " + line);
            }
        }
        Console.WriteLine();

        // ── PATTERN 3: Stop early when found ─────────────────────────
        // Find the FIRST course with price over 5000
        // Stop reading immediately — no need to scan the rest of the file
        Console.WriteLine("Pattern 3 — Stop early (first course over ₹5000):");
        using (StreamReader reader = new StreamReader(CsvPath))
        {
            reader.ReadLine();  // skip header
            string? line;
            while ((line = reader.ReadLine()) != null)
            {
                string[] parts = line.Split(',');
                if (parts.Length >= 4
                    && int.TryParse(parts[3], out int price)
                    && price > 5000)
                {
                    Console.WriteLine("  Found: " + line);
                    break;  // STOP — don't read the rest of the file
                }
            }
        }
        Console.WriteLine();

        // ── PATTERN 4: Count matching rows ───────────────────────────
        Console.WriteLine("Pattern 4 — Count Intermediate courses:");
        int intermediateCount = 0;
        using (StreamReader reader = new StreamReader(CsvPath))
        {
            reader.ReadLine();  // skip header
            string? line;
            while ((line = reader.ReadLine()) != null)
                if (line.Contains("Intermediate"))
                    intermediateCount++;
        }
        Console.WriteLine($"  Intermediate course count: {intermediateCount}");
    }
}
