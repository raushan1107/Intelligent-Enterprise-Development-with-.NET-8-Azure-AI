Structs, Enums, Collections & LINQ

You will learn how to group related data using structs, give meaningful names to fixed values using enums, store multiple items in collections, and query them with LINQ.

📦 Modules covered: 6, 7 ⏱ Duration: ~90 minutes 🛠 Tool: Visual Studio 2022 🎯 Builds on: Labs 1 & 2
MODULE 6

Structs & Enums

1 What is a struct?

A struct is a way to group a few related pieces of data together under one name. It is a value type — just like int or double — which means when you copy it you get a completely independent copy.

Use a struct when the data is small, simple, and represents a single concept — like a coordinate point, a date range, or a money amount.

Defining and using a struct — write outside your class, at the bottom of the file 
// Define the struct — bottom of Module6.cs, outside the class
struct Point
{
    public int X;
    public int Y;

    public Point(int x, int y)
    {
        X = x;
        Y = y;
    }

    public void Print()
    {
        Console.WriteLine("X = " + X + ", Y = " + Y);
    }
}

// Using it inside Run()
Point p1 = new Point(3, 7);
p1.Print();   // X = 3, Y = 7

// Accessing fields directly
Console.WriteLine(p1.X);   // 3
Console.WriteLine(p1.Y);   // 7
Output
X = 3, Y = 7
3
7
2 struct vs class — the copy difference

This is the most important thing to understand about structs. When you copy a struct, you get an independent copy. When you copy a class, both variables point to the same object.

struct — independent copy
Point p1 = new Point(3, 7);
Point p2 = p1;   // copy
p2.X = 99;

// p1.X is still 3
// p2.X is 99
// They are independent
class — shared reference
Student s1 = new Student();
s1.Name = "Raushan";
Student s2 = s1;  // same object
s2.Name = "Rahul";

// s1.Name is now "Rahul" too
// Both point to same object
Proving the difference — run this yourself 
// struct copy — independent
Point p1 = new Point(3, 7);
Point p2 = p1;
p2.X = 99;
Console.WriteLine("p1.X = " + p1.X);   // 3  — unchanged
Console.WriteLine("p2.X = " + p2.X);   // 99
ℹ

When to use struct: small data (2–4 fields), represents one concept, value-copy behaviour makes sense, no inheritance needed.
When to use class: larger or complex objects, needs inheritance, you want two variables to share the same object.

✦ Try It
  • Create a struct called CourseFee with two fields: double Amount and string Currency. Add a Print() method to it.
  • Create two CourseFee variables, copy one to the other, change the copy — confirm the original is unchanged.
3 enum — named constants

An enum is a set of named constants. Instead of writing magic numbers like 0, 1, 2 in your code, you give each value a meaningful name. This makes the code much easier to read and prevents mistakes.

Defining and using an enum — write outside your class 
// Define at the bottom of the file, outside the class
enum EnrollmentStatus
{
    Pending   = 0,
    Active    = 1,
    Completed = 2,
    Withdrawn = 3
}

// Using it inside Run()
EnrollmentStatus status = EnrollmentStatus.Active;

Console.WriteLine(status);         // Active  (prints the name)
Console.WriteLine((int)status);    // 1       (prints the number)

// Comparing
if (status == EnrollmentStatus.Active)
{
    Console.WriteLine("Student can log in");
}
Output
Active
1
Student can log in
💡

Without enums you might write if (status == 1) — but what does 1 mean? With an enum it reads like English: if (status == EnrollmentStatus.Active). Anyone reading the code immediately understands it.

4 enum in a switch statement

Enums and switch work very naturally together. The compiler will warn you if you forget to handle one of the enum values.

switch on an enum 
EnrollmentStatus status = EnrollmentStatus.Completed;

switch (status)
{
    case EnrollmentStatus.Pending:
        Console.WriteLine("Send welcome email");
        break;
    case EnrollmentStatus.Active:
        Console.WriteLine("Allow portal access");
        break;
    case EnrollmentStatus.Completed:
        Console.WriteLine("Issue certificate");   // ← this runs
        break;
    case EnrollmentStatus.Withdrawn:
        Console.WriteLine("Process refund");
        break;
    default:
        Console.WriteLine("Unknown status");
        break;
}

  • Write a switch on CourseLevel that prints the recommended duration for each level.
  • Use Enum.TryParse<EnrollmentStatus>("Completed", out var result) to convert a string to an enum value. Print the result.
5 [Flags] enum — combining values

A Flags enum lets a variable hold multiple values at the same time. Each value must be a power of 2 so they can be combined using the | operator without overlapping.

A real use case: a trainer can hold certifications in Azure and .NET and GitHub all at once.

[Flags] enum — write outside the class 
[Flags]
enum Certification
{
    None   = 0,
    Azure  = 1,    // binary: 0001
    DotNet = 2,    // binary: 0010
    GitHub = 4,    // binary: 0100
    PowerBI= 8     // binary: 1000
}

// Combine certifications with |
Certification myCerts = Certification.Azure | Certification.DotNet;
Console.WriteLine(myCerts);              // Azure, DotNet

// Check if a specific cert is held
Console.WriteLine(myCerts.HasFlag(Certification.Azure));   // True
Console.WriteLine(myCerts.HasFlag(Certification.GitHub));  // False

// Add a certification
myCerts |= Certification.GitHub;
Console.WriteLine(myCerts);              // Azure, DotNet, GitHub

// Remove a certification
myCerts &= ~Certification.DotNet;
Console.WriteLine(myCerts);              // Azure, GitHub
Output
Azure, DotNet
True
False
Azure, DotNet, GitHub
Azure, GitHub
✦ Try It
  • Create your own Flags enum called Permission with Read, Write, Delete, Admin. Give a user Read + Write, then check if they have Delete permission.
  • What happens if you forget to make the values powers of 2? Try it with 1, 2, 3, 4 instead of 1, 2, 4, 8.
MODULE 7

Built-in Collections & LINQ

1 List<T> — a resizable list

An array has a fixed size — once created you cannot add or remove items. A List<T> is like an array but it can grow and shrink. The T is a placeholder for the type — for example List<string> holds strings, List<int> holds integers.

Creating and using a List<string> 
// Create an empty list
List<string> courses = new List<string>();

// Add items
courses.Add("C# Basics");
courses.Add("Azure AI");
courses.Add("Blazor");
courses.Add("EF Core");

// Count items
Console.WriteLine("Count: " + courses.Count);   // 4

// Access by index
Console.WriteLine(courses[0]);   // C# Basics

// Loop through all items
foreach (string course in courses)
{
    Console.WriteLine(course);
}

// Remove an item
courses.Remove("Blazor");
Console.WriteLine("After remove: " + courses.Count);   // 3

// Check if an item exists
Console.WriteLine(courses.Contains("Azure AI"));   // True
Output
Count: 4
C# Basics
C# Basics
Azure AI
Blazor
EF Core
After remove: 3
True
ℹ

You can also initialise a list with values in one line:
List<string> names = new List<string> { "Raushan", "Rahul", "Priya" };

2 Dictionary<K,V> — key-value pairs

A Dictionary stores pairs of keys and values. You look up a value using its key — like a real dictionary where you look up a word (key) to find its definition (value). Keys must be unique.

Creating and using a Dictionary 
// Key = student ID (string), Value = marks (int)
Dictionary<string, int> studentMarks = new Dictionary<string, int>();

// Add entries
studentMarks.Add("S001", 85);
studentMarks.Add("S002", 72);
studentMarks.Add("S003", 91);

// Look up by key
Console.WriteLine(studentMarks["S001"]);   // 85

// Safe lookup — does not crash if key is missing
if (studentMarks.ContainsKey("S099"))
{
    Console.WriteLine(studentMarks["S099"]);
}
else
{
    Console.WriteLine("Student not found");
}

// TryGetValue — the safest way to look up
if (studentMarks.TryGetValue("S002", out int marks))
{
    Console.WriteLine("S002 marks: " + marks);   // 72
}

// Loop through all pairs
foreach (var pair in studentMarks)
{
    Console.WriteLine(pair.Key + " -> " + pair.Value);
}
Output
85
Student not found
S002 marks: 72
S001 -> 85
S002 -> 72
S003 -> 91
⚠

Accessing a key that does not exist with dict["missing"] throws a KeyNotFoundException. Always use ContainsKey or TryGetValue when the key might not be there.

  • Update a value: studentMarks["S001"] = 90; then print it — does it update correctly?
3 IEnumerable<T> — the common interface

IEnumerable<T> is the interface that List, array, and Dictionary all implement. It simply means "something you can loop through". When a method accepts IEnumerable<T> instead of List<T>, it works with any collection type.

A method that accepts any collection 
// This method works with List, array, or any other collection
static void PrintAll(IEnumerable<string> items)
{
    foreach (string item in items)
    {
        Console.WriteLine(item);
    }
}

// You can pass a List...
List<string> courseList = new List<string> { "C#", "Azure", "Blazor" };
PrintAll(courseList);

// ...or an array — same method, no changes needed
string[] courseArray = { "EF Core", "REST API" };
PrintAll(courseArray);
ℹ

You will not often write IEnumerable<T> explicitly yet — but it appears everywhere in LINQ and in method signatures in real projects. Knowing what it means will help you read real code.

4 LINQ — query syntax

LINQ (Language Integrated Query) lets you query collections using a syntax that looks similar to SQL. Instead of writing loops with if statements to filter and sort data, LINQ lets you express what you want in a single readable line.

LINQ query syntax — filter, order, select 
List<int> marks = new List<int> { 85, 42, 91, 67, 55, 78, 38 };

// Filter — only marks >= 60
var passing = from m in marks
              where m >= 60
              select m;

foreach (int m in passing)
{
    Console.Write(m + " ");   // 85 91 67 78
}
Console.WriteLine();

// Order ascending
var sorted = from m in marks
             orderby m
             select m;

foreach (int m in sorted)
{
    Console.Write(m + " ");   // 38 42 55 67 78 85 91
}
Console.WriteLine();

// Filter AND order together
var topStudents = from m in marks
                  where m >= 75
                  orderby m descending
                  select m;

foreach (int m in topStudents)
{
    Console.Write(m + " ");   // 91 85 78
}
Output
85 91 67 78
38 42 55 67 78 85 91
91 85 78
5 LINQ — method syntax

The same LINQ operations can also be written as method calls chained together. This is the style you will see most often in real .NET projects. Both do the same thing — it is a matter of preference.

LINQ method syntax — the most common style
List<int> marks = new List<int> { 85, 42, 91, 67, 55, 78, 38 }; // Where — filter var passing = marks.Where(m => m >= 60); // OrderBy — sort ascending var sorted = marks.OrderBy(m => m); // OrderByDescending var topFirst = marks.OrderByDescending(m => m); // Count — how many pass? int passCount = marks.Count(m => m >= 60); Console.WriteLine("Passing: " + passCount); // 4 // Sum, Min, Max, Average Console.WriteLine("Total : " + marks.Sum()); Console.WriteLine("Min : " + marks.Min()); Console.WriteLine("Max : " + marks.Max()); Console.WriteLine("Avg : " + marks.Average()); // Chain multiple operations together var result = marks .Where(m => m >= 60) .OrderByDescending(m => m) .ToList(); // ToList materialises the result foreach (int m in result) Console.Write(m + " "); // 91 85 78 67
Output
Passing: 4
Total : 456
Min   : 38
Max   : 91
Avg   : 65.142857142857142
91 85 78 67
💡

The m => m >= 60 part is called a lambda expression — a short anonymous function. Read it as: "for each item m, return whether m is greater than or equal to 60". You do not need to fully understand lambdas yet — just know the pattern and use it.
