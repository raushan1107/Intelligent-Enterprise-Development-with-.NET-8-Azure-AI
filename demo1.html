01
Create a New Project in Visual Studio 2022
Launch → New Project → Console App → .NET 8.0
▾
a
Open Visual Studio 2022. On the start screen, click "Create a new project".
b
In the search box type Console. Select "Console App" (the one labelled C#, Linux, macOS, Windows). Click Next.
c
Set Project name → HelloDotNet8  |  Location → a folder of your choice. Click Next.
d
On the Additional information screen, set Framework → .NET 8.0 (Long-term support). Leave "Do not use top-level statements" unchecked. Click Create.
e
VS 2022 scaffolds the project. Notice in Solution Explorer: HelloDotNet8.csproj and Program.cs.
📸 Trainer Tip: Show the "Additional Information" dialog → Framework dropdown. Students must confirm ".NET 8.0 (Long-term support)".
02
Explore the Project Structure
Solution Explorer · .csproj · Program.cs
▾
a
In Solution Explorer, double-click HelloDotNet8.csproj. Note the XML — this is the MSBuild project file.
HelloDotNet8.csproj 
<Project Sdk="Microsoft.NET.Sdk">

  <PropertyGroup>
    <OutputType>Exe</OutputType>          <!-- builds an executable -->
    <TargetFramework>net8.0</TargetFramework>  <!-- targets .NET 8 runtime -->
    <Nullable>enable</Nullable>           <!-- C# nullable reference types -->
    <ImplicitUsings>enable</ImplicitUsings> <!-- auto-imports System, etc. -->
  </PropertyGroup>

</Project>
b
Now open Program.cs. You will see the auto-generated default file — just one line! This is C# 9+ Top-Level Statements.
Program.cs — Auto-generated (default) 
// See https://aka.ms/new-console-template for more information
Console.WriteLine("Hello, World!");
💡 Key point: No class Program, no static void Main. The compiler wraps this in a hidden entry point automatically.

Write real code: variables, interpolation, loops, user input
▾
a
Replace the entire content of Program.cs with the code below. Type it manually — do not paste — to build muscle memory.
Program.cs — Your Version 
// ─────────────────────────────────────────────────
//  Demo 1 · Module 1 — .NET 8 & Azure AI Training
//  HelloDotNet8 · Program.cs
// ─────────────────────────────────────────────────

// 1️⃣  Basic output
Console.ForegroundColor = ConsoleColor.Cyan;
Console.WriteLine("═══════════════════════════════════");
Console.WriteLine("  Welcome to .NET 8 — Demo 1        ");
Console.WriteLine("═══════════════════════════════════");
Console.ResetColor();

// 2️⃣  Variables and string interpolation
string courseName = ".NET 8 & Azure AI";
int    totalDays  = 11;
double version   = 8.0;

Console.WriteLine($"Course  : {courseName}");
Console.WriteLine($"Duration: {totalDays} days");
Console.WriteLine($"Runtime : .NET v{version}");
Console.WriteLine();

// 3️⃣  User input
Console.Write("Enter your name: ");
string? name = Console.ReadLine();
Console.WriteLine($"Hello, {name}! Let's build something awesome.\n");

// 4️⃣  Simple loop — print module days
Console.WriteLine("── Course Modules ──");
string[] modules =
[
    "Writing Applications in C# & .NET 8",
    "OOP Fundamentals",
    "Collections & LINQ",
    "Async & Parallel Programming",
    "Azure AI Integration"
];

for (int i = 0; i < modules.Length; i++)
{
    Console.WriteLine($"  [{i + 1}] {modules[i]}");
}

// 5️⃣  Environment information (runtime introspection)
Console.WriteLine();
Console.WriteLine("── Runtime Info ──");
Console.WriteLine($".NET Version : {Environment.Version}");
Console.WriteLine($"OS Platform  : {Environment.OSVersion}");
Console.WriteLine($"Machine Name : {Environment.MachineName}");
Console.WriteLine($"64-bit OS    : {Environment.Is64BitOperatingSystem}");

Console.WriteLine("\n✅ Program completed successfully.");
b
Notice IntelliSense auto-completing as you type. Hover over Console to read the tooltip — VS 2022 shows the full XML docs inline.
c
Observe the array initializer syntax on line with modules — this is C# 12 collection expressions using [ ] instead of new string[]{ }.

05
Understand the Concepts
CLR · JIT · IL · Top-level Statements · Implicit Usings
▾
⚙️
CLR — Common Language Runtime
The virtual machine of .NET. Manages memory (GC), type safety, exception handling, and JIT compilation. Your code never runs directly — the CLR runs it.
Source → IL → JIT → Native
🔥
JIT — Just-In-Time Compiler
.NET 8 uses RyuJIT. Converts IL bytecode to native CPU instructions at runtime. First run compiles; subsequent calls use the cached native code. .NET 8 adds Tiered Compilation for hot paths.
IL Bytecode → x64 Native
📝
Top-Level Statements
C# 9+ feature. No boilerplate class/Main needed. The compiler silently wraps your code in a <Program>$ class with a synthesized entry point. Cleaner, less ceremony.
// compiler generates: static void Main(string[] args) { // your top-level code }
📦
Implicit Usings
Enabled by ImplicitUsings in .csproj. A global using file is auto-generated: System, System.IO, System.Linq, System.Threading.Tasks, etc. are always available — no explicit using needed.
// auto-injected globally: global using System; global using System.Linq; global using System.IO;
🏛️
.NET 8 Ecosystem
LTS release (3 years support). Unifies .NET Framework, .NET Core, Xamarin into one platform. Ships with C# 12, improved performance, Native AOT, and built-in AI/ML abstractions.
dotnet --version // 8.0.xxx
📁
Project Structure
MSBuild (.csproj) is the build engine. The SDK-style project is minimal by default. NuGet packages, build targets, and publish profiles are all configured here.
HelloDotNet8/ Program.cs HelloDotNet8.csproj bin/ ← output obj/ ← intermediate

🚀 Extend HelloDotNet8 — Pick Any 2

Add a while loop that keeps asking for names until the user types exit.
Use DateTime.Now to print a greeting: "Good morning, Raushan!" or "Good evening, Raushan!" depending on the current hour.
Create a local function called PrintBanner(string title) and call it at the top of Program.cs.
Use Environment.GetCommandLineArgs() to print all CLI args, then run with dotnet run -- arg1 arg2.
Add a try { } catch (Exception ex) { } block around the user input section. Test what happens when you press Ctrl+C mid-input.
Bonus: Open IL DASM (Developer Tools → IL Disassembler) or install ILSpy extension and inspect the hidden <Program>$ class that wraps your top-level code.
Hint — Local Function Pattern 
// Local function — defined inside top-level scope
void PrintBanner(string title)
{
    int width = title.Length + 4;
    Console.WriteLine(new string('─', width));
    Console.WriteLine($"  {title}  ");
    Console.WriteLine(new string('─', width));
}

// call it at the top
PrintBanner("HelloDotNet8 · Demo 1");
